# Exp_M · M×ΔV writer 实验(最终,80 题)

> **80 题**全 12 分支已 kimi-k2.5 打分(deepsearchqa 官方口径)。这是 Exp_M 的最终快照,取代中期版 [comparison_vs_14branch_interim.md](comparison_vs_14branch_interim.md)。
> 分支:1 full prefill(上界)、3 full reuse(下界)、ΔV/ΔK×{token,layer}@50%、**M×ΔV×{token,layer}×{75,50,25}**(主方案)。
> 实验设计见 [experiment_design.md](experiment_design.md);M 怎么算/对齐见 [M_pipeline_explained.md](M_pipeline_explained.md)。

---

## 0. 最终结论(两句话)
1. **主方案 M×ΔV 是 null**:相对纯漂移 ΔV/ΔK(同 50% 同粒度,Exp_M 内部干净对比),M×ΔV 的差全在 ±0.03、**CI 全跨 0、一个都不显著**。"用 supervisor 注意力加权选位优于纯漂移"**没拿到任何证据**。
2. **而且这一轮整体信噪比被压扁,实验本身欠功效**:新 run 的 **full-prefill 上界只有 31.1**,比旧 14分支实验在**同样 80 题**上的 **35.6 低了 ~4.5 分**;于是连最基本的"复用伤害"(prefill−reuse)都只剩 **+0.033、不显著**(旧 run 同题是 +11.0 分、清清楚楚)。**上界没拉开 → 所有选择性重算信号都被噪声淹没。**

> 一句话:**这轮没能证明 M×ΔV 有用,但更可能是"新 run 报告整体偏弱、动态范围太窄"导致谁都测不出,而非确证 M×ΔV 无效。** 见 §4 的根因分析。

## 3. 五条发现
1. **M×ΔV null**:相对纯漂移无优势(§2B 50% 行全不显著、效应 ≤0.03)。
2. **"选层 vs 选 token"也 null**:三挡都 ≈0(§2C)。早先苗头是小样本噪声。
3. **复用伤害在新 run 没显著**(+0.033,§2A)——与旧实验(同题 +11 分)强烈反差。
4. **根因 = 新 run 上界塌 4.5 分**(prefill 31.1 vs 旧 35.6,§1),动态范围被压到 3.3 分,信噪比不够测任何东西。
5. **机制侧本身是对的**:M 抓取、ΔV/ΔK、transfer_M 对齐这轮全部验证正确(见 [verification_method.md](verification_method.md)、[M_pipeline_explained.md](M_pipeline_explained.md))——所以这个 null **不是 M 算错**,而是实验功效不足。

---

## 4. 根因:新 run 为什么整体偏弱(下一步该查这个)
**同样 80 题、同样"writer 全重算 summary",新 run 的 full-prefill 比旧 run 低 4.5 分(31.1 vs 35.6)。** 两者唯一差别是**喂给 writer 的 summary 来自不同 demo run**:
- 新 run 强制 **supervisor 全复用 summary KV**(为抓 M 决策注意力而设);
- 旧 run 是普通 CacheBlend demo。

**假设**:supervisor 全复用可能让上游决策/检索走偏 → 产出的 summary 质量打折 → 连"全重算"的 writer 也救不回来 → 整体分数低、范围窄。**这本身是个值得单独验证的现象**(也呼应 [[supervisor-reuse-harmless-and-judge-lesson]] 当时"只 supervisor 复用对 report 无害"的结论需要在这个更强设定下复查)。

**建议下一步(二选一)**:
- **A. 先修信噪比再下结论**:换成"supervisor 也走普通 CacheBlend(非强制全复用)"重跑,把上界拉回 ~35,再看 M×ΔV 有没有用——避免"上界塌"掩盖真实信号。
- **B. 接受 null**:若认为强制全复用是必须的设定,则结论就是"在该设定下 M×ΔV 无可检出收益",收尾。

- **null ≠ 证伪**:80 题、噪声底 0.14,而本轮所有效应 ≤0.03 且范围被压扁 → 是"**测不出**",不是"**确定无效**"。要干净结论得先解决 §4 的上界问题。
