# 增益拆解 + 过拟合诊断

← 返回 [experiment-data-summary.md](../experiment-data-summary.md)

## 增益三级拆解(架构 / workload / 域训)

DRGym 上从"未域训起点"到"二期"的完整台阶,把三种增益分开(均为干净 held-out 加速比):

| 台阶 | 加速比 | 增量 | 归因 |
|---|--:|--:|---|
| vanilla | 1.00× | — | 基准 |
| **AngelSlim(未域训)on DRGym** | **1.071×** | +0.07× | **架构红利**(EAGLE 学习式 draft 在长 prompt 就已翻正、小胜 suffix) |
| → 一期 8k-2ep | 1.230× | +0.16× | **域训效应(窄)** |
| → 二期 12k-3ep | 1.303× | +0.07× | **域训效应(强,监督×2+12288+3ep)** |
| **域训累计(1.071→1.303)** | | **+0.23×** | **决定性因素** |

对照 suffix 0.976×(净减速):**未域训 AngelSlim 1.071× 就已超过 suffix**,但只勉强翻正;域训把它推到 1.303× 的碾压。

## workload 效应(跨数据集,用同一 head 对比)

同一未域训 AngelSlim,DRBench vs DRGym:

| head | DRBench(中文短) | DRGym(英文长) | workload 增量 |
|---|--:|--:|--:|
| AngelSlim 加速比 | 0.94×(净减速) | 1.071× | **+0.13×** |
| AngelSlim 接受率 | 11.1% | 22.8% | **×2.05** |
| 一期 8k 加速比 | 1.10× | 1.230× | +0.13× |
| 一期 8k 接受率 | 17.6% | 34.2% | ×1.94 |

**长 prompt 让同一 head 接受率翻倍**(11.1→22.8%、17.6→34.2%),加速比 +0.13×——这是纯 workload 效应(head 没变)。

## 关键洞察(三句话)

1. **架构是必要条件**:未域训 AngelSlim 在长 prompt 就翻正超 suffix(1.071× vs 0.976×),证明"长 prompt 利好 EAGLE"部分源于架构本身;
2. **workload 是放大器**:同一 head 换到长 prompt,接受率翻倍(+workload 效应 +0.13×);
3. **域训是决定性因素**:架构红利只 +0.07×(勉强超 suffix),域训 +0.23× 才把"勉强赢"变成"碾压"。

对照 DRBench 上域训边际收益枯竭(一期→二期 +0.8pp,见 [02](02-drbench-short-prompt.md)):**域训的价值本身也依赖 workload——长 prompt 上域训还大有可赚,短 prompt 上已到顶**。

## 过拟合诊断(干净 held-out vs 训练集内)

二期 12k-3ep:干净 1.303× vs 训练集内 1.339×,**gap 仅 +0.036×**。

一期 8k-2ep:干净 1.231× vs 训练集内 1.233×,**gap 仅 +0.002×**。

**结论**:两代模型泛化均良好,训练集内偏高极轻。干净样本的加速比是可信的泛化性能(非过拟合虚高),故 [03](03-drgym-long-prompt.md) 的"EAGLE 碾压 suffix"结论建立在公平的干净数据上。
