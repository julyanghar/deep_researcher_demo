# Exp A — 单次复用的 KV 漂移结构与决策归因(完整实验计划)

> **这份文档是什么**：Exp A 的完整、可独立执行的实验计划。读完不需要回看对话即可动手。
>
> **Exp A 一句话**：在一条用 full prefill 跑出来的「金标准轨迹」上,把每个 summary 段 $s_i$ 的 KV 在「复用」和「真值」两种算法下各算一次,用 **teacher forcing 锁死 token 序列**,逐层测各种 KV 偏差,再回归找出**哪一种偏差最能预测下游决策伤害**——这个量就定义为后续 gate 要用的 $\Delta$。
>
> **Exp A 不回答什么**:它不测「轨迹分叉多少 / 报告差多少」(那是 Exp B)。Exp A 测的是**单次复用的 KV 偏了多少、以及偏差如何一路传到决策**,要求两条轨迹逐 token 对齐、可配对。

---

## §1 实验目标与核心判断

**目标**:复用 KV 偏离真值时,到底是 **K 还是 V、方向还是幅度、哪一层**最能预测「下游决策伤害」?

**为什么必须先做这个**:如果你直接拿某个偏差量(比如照抄 RelayCaching 的「V 余弦」)去 gate,效果不好时你**无法归因失败**——是这个量选错了?还是组合方式错了?还是伤害根本不存在?三者缠在一起。Exp A 的全部价值是**在引入任何加权/组合之前,先把「哪个 KV 偏差量致害」这一个因素隔离出来钉死**。这和你证伪 head 级选择用的是同一套方法论:先隔离单因素,再谈组合。

**产出**:一个被数据选出来的 $\Delta$ 定义(用哪个 K/V 几何量、聚合到哪些层),直接喂给 Phase 2 的刷新 gate。附带一个可能反 RelayCaching 的实证发现(如果致害量不是 V 余弦)。

> **配置说明(与「用不用 `r_t`」正交)**:Exp A 复用的对象是 summary 段 $s_i$,不是 reasoning `r_t`。supervisor 上下文里加不加逐轮 `r_t` 都不改变 Exp A 的结构——加了的话 `r_t` 就是 teacher-forced 轨迹里固定前缀的一部分,且后面采集 $M$ 时从 `r_t` 的多个 token 读;不加则 $M$ 只从决策 token 读。Exp A 本体不受影响。

## §2 核心设计原则:teacher forcing 控制变量

### 2.1 为什么不能让 reuse 版自由跑

朴素做法是「full 版跑一遍、reuse 版跑同样题目、对比」。**这在第二步就崩**:reuse 会扰动决策 → 决策可能翻转 → 从翻转那一刻起两版检索不同文档、生成不同 summary、走进不同轮次。于是你想逐 token 比较 KV 偏差时,**两条轨迹的 token 根本对不上**——full 版第 3 轮的 $s_3$ 和 reuse 版第 3 轮的 $s_3$ 是不同文档的不同摘要,逐 token 相减毫无意义。

根因:你把两个冲突的诉求塞进了一次对比。「测 KV 偏差」需要**两条轨迹完全对齐**(同 token);「测决策是否翻转」恰恰**允许轨迹分叉**。这俩不能在同一次 run 里同时满足。

### 2.2 控制变量:只让「KV 来源」这一个变量动

铁律是一次只动一个变量。要测的因果是「**KV 复用** → 决策偏差」。teacher forcing 把 token 这个变量摁住(强制两版读完全相同的 $s_1…s_t$ 文本),就只剩 KV 来源在动,logits 的差才能干净归因到「复用」这一件事。

> **类比**:测「某肥料让番茄长得不同」,得让两块地的种子、水、光照全相同,只有肥料不同。teacher forcing = 锁死种子/水/光照,只留肥料(KV 来源)这一个变量。reuse 自由跑 = 两块地什么都不一样,长出来不同你啥也证明不了。

**具体操作**:同一条金标准轨迹的 token 序列,跑两遍前向——
- **真值模式**:$s_i$ 用 full prefill 在 supervisor 上下文里算出的真值 KV $\mathrm{KV}^*(s_i)$。
- **复用模式**:$s_i$ 用 worker 那边算的 $\mathrm{KV}^w(s_i)$ 经 RoPE 对齐得到的 $\mathrm{KV}^r(s_i)$。
- 其余完全一致,在决策 token 位比两遍的 logits。

所以 **Exp A 里 reuse 不是一个会自由分叉的 agent,而是一个逐 token 的 KV 替换操作**。

### 2.3 Exp A 与 Exp B 的分工(别混)

| | Exp A | Exp B |
|---|---|---|
| teacher forcing | **开**(锁死轨迹) | 关(自由跑) |
| 轨迹 | 两版逐 token 对齐、可配对 | 允许分叉、不配对 |
| 测什么 | 单次复用的 KV 偏差 → 决策 KL | 决策翻转后轨迹发散、报告质量差 |
| 归因 | 干净(只动 KV 来源) | 测发散(看复用整体后果) |

## §3 符号定义

| 符号 | 含义 |
|---|---|
| $P_0$ | 固定前缀 = system prompt + 原始 query |
| $s_i$ | 第 $i$ 轮 worker 产出的 summary 段(被复用对象) |
| $r_t$ | supervisor 第 $t$ 轮 decode 出的 reasoning + 决策 token(可选) |
| $C^w_i=[P_w,\text{doc}_i]$ | worker 上下文,$s_i$ 在此生成 |
| $\mathrm{KV}^w(s_i)$ | $s_i$ 在 worker 上下文里的 KV(**复用源**) |
| $C^s_t=[P_0,s_1,\dots,s_t]$ | 第 $t$ 轮 supervisor 上下文(append-only) |
| $\mathrm{KV}^*(s_i)$ | $s_i$ 在 $C^s_t$ 里完整 prefill 的**真值 KV**(刷新要逼近的靶子) |
| $\mathrm{KV}^r(s_i)$ | $\mathrm{KV}^w(s_i)$ 经 RoPE 对齐到新位置的**复用 KV** |
| $K^r_{l,h,j},V^r_{l,h,j}$ | 复用 KV 在层 $l$、头 $h$、token $j$ 上的向量 |

| $M_i^{(t)}$ | 第 $t$ 轮 decode token 对 $s_i$ 的入向注意力(Phase 1 用,Exp A 顺手采集) |
| $L,H,J$ | 层数 / KV 头数 / token 数 |

## §4 测量对象:七条候选预测器 + 一个金标准

### 4.1 三个空间层次

KV 偏差到决策之间隔着 softmax 和几十层网络,是个非线性黑箱。**KV 偏差大不等于决策偏差大**——某些方向被 softmax 放大,某些被压平。所以要测一整条链,越往下越接近你真正在乎的伤害:

```
K/V 几何偏差(余弦/norm/欧氏)   ← 输入空间,最易测,离伤害最远
        │ 过 softmax(非线性)
        ▼
注意力输出偏差(第7条)          ← 半程,过了非线性
        │ 过后续所有层 + unembedding
        ▼
决策 logits KL                  ← 金标准 = 你定义的伤害本身
```

### 4.2 七条候选预测器(自变量)

K 和 V 各 3 条:**方向(余弦)、幅度(norm)、合体(欧氏)**。设逐向量比较:

- **余弦偏差(方向)**:$d_{\cos}(v^r,v^*)=1-\dfrac{v^r\cdot v^*}{\|v^r\|\,\|v^*\|}$,只看夹角,无视长度。
- **norm 偏差(幅度)**:$d_{\text{norm}}(v^r,v^*)=\dfrac{\big|\,\|v^r\|-\|v^*\|\,\big|}{\|v^*\|}$,只看各自长度,无视方向。
- **欧氏偏差(合体)**:$d_{\text{euc}}(v^r,v^*)=\|v^r-v^*\|$,逐元素相减,**同时含方向和幅度**。

| # | 曲线 | 空间·维度 | 角色 |
|---|---|---|---|
| 1 | K 余弦 | 输入·方向 | 主候选 |
| 2 | K norm | 输入·幅度 | 主候选 |
| 3 | K 欧氏 | 输入·合体 | **对照基线** |
| 4 | V 余弦 | 输入·方向 | 主候选 |
| 5 | V norm | 输入·幅度 | 主候选 |
| 6 | V 欧氏 | 输入·合体 | **对照基线** |
| 7 | 注意力输出偏差 | 半程(过 softmax) | 中间观察量 |

**欧氏(3、6)为什么是对照基线而非主判据**:它糊了方向和幅度,留着是为了回答「**把方向/幅度拆开到底有没有必要**」——如果余弦单独就和欧氏一样能预测伤害(norm 没用),证明伤害纯由方向驱动,该丢欧氏只用余弦(更省、更可解释);如果欧氏和「余弦+norm 一起」打平,说明拆开没额外信息,直接用欧氏更简。跑完 Exp A 欧氏大概率被淘汰,但必须在场来证明拆分有无必要。**直接用欧氏当主判据正是 CacheBlend 踩的坑**(它用幅度类判据选 token,真实漂移在方向上,选错了)。

**第 7 条为什么单列、不能并进 K/V**:$o=\mathrm{softmax}(qK^\top)V$ 里,K 偏差改的是 softmax **权重**(注意力分配),V 偏差改的是被加权的**内容**,两者穿过 softmax 会**非线性交互**(K 把注意力挪到某 token、V 又恰好在那 token 上偏,叠加放大或抵消)。这种交互单看 K 欧氏或 V 欧氏都看不出,只有一起喂进 $\mathrm{softmax}(qK^\top)V$ 才显形。

### 4.3 金标准:决策 logits KL(不是曲线,是回归靶子)

**这是评判前 7 条的尺,本身不参与「被预测」。** 前 7 个在竞争「谁最能预测决策 KL」。

精确定义(**这里有个要害,别写错**):是在 **decode 的决策 token 位置**,取**过完所有层 + 过完 unembedding/LM head 投影到词表**后的概率分布算 KL——**不是「最后一层 hidden state」**。

- 「最后一层」输出是 $d_{\text{model}}$ 维(如 4096)的 hidden state,**还不是 logits**;要再过 unembedding 矩阵 $[d_{\text{model}}\times \text{vocab}]$ 才投影成词表大小(如 128k)的 logits,softmax 后才是「下一个 token 是谁」的概率。
- KL 散度定义在**概率分布**上,只能在 vocab 维概率上算,**不能在 hidden state 上算**(对 4096 维 hidden state 算 KL 是无意义的)。

$$D_{\mathrm{KL}}(p^*\,\|\,p^r)=\sum_{v\in\text{vocab}} p^*(v)\,\log\frac{p^*(v)}{p^r(v)}$$

其中 $p^*$ = 真值模式在决策位置的词表分布,$p^r$ = 复用模式的。它是金标准,正因为它就落在模型**真正做决策**的语义上(词表概率),而非某中间层内部表示。

### 4.4 各曲线的测量粒度(容易搞混)

- **曲线 1–6(K/V 几何)**:底层数据粒度是 **每层 $l$ × 每 summary token $j$**(对 KV 头取平均)。同一份底层数据按目的聚合:
  - 逐 token 曲线(看哪些 token 漂):$\text{dev}(j)=\frac{1}{L\cdot H}\sum_{l,h} d(\cdot)$,每 token 一个数。
  - 逐 layer 曲线(看 U 形):$\text{dev}(l)=\frac{1}{J\cdot H}\sum_{j,h} d(\cdot)$,每层一个数。
  - 喂回归(见 §6):聚合到「summary 事件级」特征,不保留逐 token 行(原因见 §6,避免伪重复)。
- **曲线 7(注意力输出偏差)**:粒度是 **每层 × 每个 decode query**(对整段 $s_i$ 的 KV 汇总成一个输出向量),**不是每 summary token**——单个 token 的偏差已被 1–6 测了。$q$ = supervisor 决策时刻 decode 的 token 的 query。
- **GQA 模型(Llama-3.1、Qwen2.5)**:K/V 头数远少于 Q 头数,上面所有 $H$ 用 **KV 头数**,别误用 Q 头数。

### 4.5 多轮场景:每个量记几次(按轮不变 vs 按轮变化)

多轮 loop 里 summary 累积:$s_i$ 在第 $i$ 轮插入,之后第 $i, i{+}1, \dots, T$ 轮都被读。**对每一轮的 supervisor,KV 偏差、注意力输出偏差、决策 KL 都要重记吗?** 答案按量分裂,而且这个分裂本身揭示了 M×Δ 为什么必要。

**核心事实:append-only 固定顺序 → 每个 $s_i$ 的 KV 跨轮不变。** $s_i$ 一旦插入,前缀 $[P_0,s_1\dots s_{i-1}]$ 就冻住;后续轮新加的 $s_{i+1}\dots s_t$ 都排在 $s_i$ 后面,**因果掩码下 $s_i$ 根本不 attend 它们**。所以真值 $\mathrm{KV}^*(s_i)$ 和复用 $\mathrm{KV}^r(s_i)$ 在第 $i$ 轮算出后,第 $i{+}1\dots T$ 轮**逐 bit 相同**(这正是 prefix caching 免费的原因;加不加 `r_t`、`r_t` 是否交错都不影响,只要 append-only)。

推论:**KV 几何偏差 $\Delta_i$(曲线 1–6)只随 $s_i$ 变、不随轮变 → 每个 summary 只测一次,每轮重测是纯冗余。**

但注意力和决策是另一回事——它们依赖「这一轮在 decode 什么」:

| 量 | 随轮变? | 为什么 | 记几次 |
|---|---|---|---|
| KV 几何偏差 $\Delta_i$(曲线1-6) | **否** | append-only 冻住 $s_i$ 前缀,$\mathrm{KV}^*(s_i)$ 跨轮不变 | **每 summary 一次** |
| 注意力 $M_i^{(t)}$ | 是 | 每轮 reasoning/决策对 $s_i$ 的关注度不同 | 每轮 |
| 注意力输出偏差(曲线7) | **是** | $o=\mathrm{softmax}(qK^\top)V$ 里 query $q$ 是该轮决策的、rest 段也随轮增长 | 每个决策点 |
| 决策 logits KL | 是 | 决策基于该轮上下文 $[P_0,s_1\dots s_t]$,逐轮不同 | 每个决策点 |

**直接回答你的问题:KV 偏差不用每轮记(每 summary 记一次就够,跨轮恒定);注意力输出偏差和决策 KL 要按轮记**,因为它们随该轮的 decode query $q$ 和该轮的决策而变。

**这个分裂正是 M×Δ 的实证种子(别在 Exp A 里强行解决,留给 Phase 1)**:同一个 $s_i$、同一个固定的 $\Delta_i$,在不同轮造成的伤害**不一样**——差别全在该轮对 $s_i$ 的注意力 $M_i^{(t)}$ 上。某轮决策几乎不看 $s_i$(低 $M$)→ 哪怕 $\Delta_i$ 很大也几乎不致害;某轮决策重度依赖 $s_i$(高 $M$)→ 同样的 $\Delta_i$ 造成大伤害。这意味着**伤害 $\approx \Delta \times M$,光靠 $\Delta$ 预测不全**。所以多轮不是要你多测 $\Delta$,而是它恰好暴露了「为什么需要把 $M$ 乘进来」——这条观察直接喂给 Phase 1 的 Exp D。

**对 Exp A 归因回归的影响(重要)**:因为 $\Delta_i$ 跨轮不变、伤害却随轮变,如果你把所有 $(s_i, \text{第 } t \text{ 轮})$ 对都塞进回归、用 $\Delta_i$ 当特征,会看到「同一个 $\Delta_i$ 散在一堆不同伤害值上」→ 相关被 $M$ 的变动冲淡、且 $\Delta_i$ 特征在多行间重复。**干净做法:Exp A 的「哪个 $\Delta$ 形态最致害」回归,每个 summary 只锚定到一个轮——它的插入轮 $i$**(刚插入、自然被高度关注,$M$ 较饱满),得到一个独立的 $(\Delta_i, \text{伤害}_i)$ 事件。这样既避免特征重复,又把 $M$ 这个混淆变量近似控住,让 Exp A 专注回答它该回答的「哪个几何量是对的 $\Delta$」。同一 $s_i$ 在后续轮的伤害变化,作为「$\Delta\times M$」的观察证据**另行记录**,不进这个回归。

## §5 完整实验步骤

### Step 0｜搭 teacher-forcing 测量框架(地基)

能力要求:给定一条固定 token 序列 $[P_0,s_1,\dots,s_t]$,supervisor 能在两种模式下前向——(a) 全真值 prefill;(b) 指定 $s_i$ 段用复用 KV(RoPE 对齐)替换,其余真值。两种模式吃**同一串 token**。

- 服务栈:`vLLM` + `LMCache` + `CacheBlend`。
- **读 attention 的坑**:`vLLM` 默认 FlashAttention 融合核**不物化注意力权重**,拿不到逐 token attention。测量阶段用 HF eager attention(`output_attentions=True`)或给 vLLM 打 patch——跑慢但离线测量够用。
- 检索后端:用**冻结的 DDG/Brave + 两级缓存**(`search_cache[query]→URL列表` + `page_cache[url]→内容`,frozen 只读模式)跑出金标准轨迹,保证轨迹可复现。注意:Exp A 一旦记下轨迹的 token 序列,后续就是**重放记录的 token**,基本不再碰检索。
- 这套地基 Exp B/G 都要复用,值得一次搭好。

### Step 1｜跑 full prefill,建金标准轨迹 + 全档案

- 固定题目集 **≥ 30 题**,覆盖不同轮深 $T$。
- 每题每轮存:检索文档、$s_i$ 的 token 序列、supervisor 上下文、**决策 token 位置的完整词表 logits 分布**(⚠ 不是 argmax,存整个分布,否则 KL 算不了)。
- **顺手 dump**:supervisor decode 时 decode token 对每个 $s_i$ 的入向 attention $M_i^{(t)}$——这是 Phase 1 的信号,同一趟采集,不必为 Phase 1 重跑(概念分层 ≠ 采集分趟)。
- **记录节奏(见 §4.5)**:logits、$M_i^{(t)}$、注意力输出偏差是**按轮**记的(随该轮决策而变);而下一步要配的 KV 偏差 $\Delta_i$ **每 summary 只记一次**(append-only 下跨轮恒定),不必每轮重算。
- 模型:`Llama-3.1-8B-Instruct` 为主,`Qwen2.5-7B` 做跨模型复核。

### Step 2｜在金标准轨迹上,对每个 $s_i$ 做「复用 vs 真值」配对

- 对每个被复用的 $s_i$:取 $\mathrm{KV}^w(s_i)$ 做 RoPE 对齐 → $\mathrm{KV}^r(s_i)$;同位置真值 $\mathrm{KV}^*(s_i)$ 就是 Step 1 存的。
- ⚠ **RoPE 对齐必须先验证正确**:把 worker 里位于位置 $p$ 的 key 旋到 supervisor 里新位置 $p'$(乘 $R_{\Theta,\,p'-p}$)。**验证法**:在不该有内容差异的纯前缀位置上测偏差,应 $\approx 0$;若不为 0 说明对齐错了,后面测的全是位置错配伪影而非内容漂移。这是 PIC 家族(CacheBlend/EPIC/KVLink)的共同地基。
- ⚠ **锁同模型**:supervisor/worker 同骨干,KV 语义空间一致,复用只差位置(RoPE)。跨模型还要处理表示空间对齐,是另一回事,Exp A 先不碰。

### Step 3｜算 K/V 几何偏差(曲线 1–6,描述性)

- 对每个 $s_i$ 的每 token、每层,算六个量(K/V × 余弦/norm/欧氏),对 KV 头取平均。
- 出**逐 token 曲线**和**逐 layer 曲线**(各画 6 条)。看:漂移是否稀疏(少数 token 漂)?逐层是否 U 形(浅层稳、中层狠)?
- 这一步是**理解漂移结构**的可视化,还不做归因。

### Step 4｜算偏差如何穿过网络(曲线 7 + 金标准)

**曲线 7,注意力输出偏差,两个版本**:

- **隔离版(测单层敏感度,做 oracle)**:某层,只把 $s_i$ 的 KV 在复用/真值间换,其余段锁真值,$q$ 用真值的:
$$o^*=\mathrm{softmax}\!\Big(\tfrac{qK^{*\top}}{\sqrt d}\Big)V^*,\quad o^r=\mathrm{softmax}\!\Big(\tfrac{q[K^*_{\text{rest}};K^r_{s_i}]^\top}{\sqrt d}\Big)[V^*_{\text{rest}};V^r_{s_i}]$$
$$\text{dev}(l)=\frac{\|o^r-o^*\|}{\|o^*\|}$$
逐层画,看哪层的 $s_i$ KV 偏差最伤输出。因 rest 和 $q$ 锁真值,**只归因到 $s_i$ 这一层、不含跨层复合** → 这是「该不该修这一层」的判据。
- **复合版(端到端)**:不锁 rest、不锁 $q$,整条复用前向自然跑,误差逐层复合。它实际上并入了决策 KL(链路终点)。隔离版与复合版之差 = 误差跨层复合量。

**决策 logits KL,金标准**:决策时刻,真值模式与复用模式各前向一遍(同上下文),取词表分布算 $D_{\mathrm{KL}}(p^*\|p^r)$(定义见 §4.3)。

⚠ **决策 token 怎么界定**:JSON 输出时「决策」不是整个 JSON 对象,而是真正决定 continue/handoff 的那个关键 token 位。比如输出 `{"action": "continue"}`,就在区分 continue/handoff 的那个 token 位置取词表分布算 KL,**不要**对整个 JSON 每 token 都算。锁定到「决策落子」那一位,KL 才对应「决策偏了多少」。

### Step 5｜归因回归:找出最能预测决策伤害的量(见 §6 详解)

- **swap-one-summary 设计**:为把 KL 干净归因到某个 $s_i$,把所有 summary 锁真值、**只换 $s_i$ 一个**为复用,测决策 KL → 这个 KL 可归因于 $s_i$。每个 $(s_i,\text{决策})$ 对各一次。
- 把 $s_i$ 的逐 token 逐层偏差**聚合成事件级特征**,回归预测该 $s_i$ 的归因 KL。
- 产出:哪个 K/V 几何量(方向/幅度、K/V、哪些层)相关最高且最便宜 → **定为 $\Delta$**。

## §6 归因方法:事件级回归(大白话 + 修正伪重复)

### 6.1 为什么不能只看首尾

你手上有起点(KV 偏了多少,每层每 token 一个数)和终点(决策 KL,一个数)。只有这两端,你只能问「KV 偏得多决策就偏得多吗」,**答不出「哪种偏、哪一层的偏才真正传到决策」**——中间是黑箱。归因不靠理解黑箱内部,靠**统计回归**直接告诉你哪个起点变量和终点最相关。

### 6.2 回归的单位:一个「复用事件」,不是一个 (token, layer)

⚠ **这里修正一个常见错误设计**。直觉上想把表格做成「每行一个 (s_i, token, layer),最后一列填该 $s_i$ 的决策 KL」——但同一个 $s_i$ 的几百个 (token,layer) 行会**共享同一个 KL 值**,这叫**伪重复(pseudoreplication)**:行不独立、虚高样本量、相关系数被夸大,审稿人会抓。

**正确单位 = 一个复用事件 = 「某个 $s_i$,锚定到它的插入轮」**(见 §4.5:$\Delta_i$ 跨轮恒定,所以每 summary 只锚一个轮,否则 $\Delta_i$ 特征会在多轮间重复、且被 $M$ 的变动冲淡)。每个事件一行:

| 事件(行) | K余弦_mean | K余弦_max | V余弦_mean | V余弦_max | …(各量×聚合) | …按层分段的均值 | **归因决策KL** |
|---|---|---|---|---|---|---|---|
| 题1·s_1(插入轮) | 0.04 | 0.21 | 0.33 | 0.71 | … | … | **0.05** |
| 题1·s_2(插入轮) | 0.02 | 0.09 | 0.08 | 0.18 | … | … | **0.002** |
| 题2·s_1(插入轮) | 0.06 | 0.30 | 0.55 | 0.88 | … | … | **0.11** |
| … | … | … | … | … | … | … | … |

- **左边特征(自变量)**:把该 $s_i$ 逐 token 逐层的偏差**聚合**成几个标量——每个量的 mean 和 max(max 抓「最坏的那个 token/层」往往比 mean 更有判别力),外加**按层分段的均值**(浅/中/深层各一个,用来查哪段层致害)。
- **最右(因变量)**:swap-one-summary 在该 $s_i$ 插入轮测出的、可归因于它的决策 KL。
- **行数**:30 题 × 每题数个 summary ≈ 几百到上千个**独立**事件(每 summary 一行,不按轮膨胀)。比伪重复的「几十万行」少,但每行独立,统计才站得住。
- **后续轮的数据不丢**:同一 $s_i$ 在 $i{+}1\dots T$ 轮是「同 $\Delta_i$、不同伤害」,这些**另作 $\Delta\times M$ 的证据**(§4.5),喂给 Phase 1 验「伤害 $\approx\Delta\times M$」,但**不进这张 $\Delta$-形态回归表**。

### 6.3 回归在干嘛(大白话)

逐列问「**这一列**(比如 V余弦_max)和**最右列**(归因 KL)关系多大」:

```
V余弦_max   ↔ 决策KL :  相关 0.7   ← 最高,留它当 Δ
V欧氏_max    ↔ 决策KL :  相关 0.6
K余弦_max   ↔ 决策KL :  相关 0.3
V norm       ↔ 决策KL :  相关 0.1   ← 没用
K norm       ↔ 决策KL :  相关 0.05  ← 没用
```

系数最高的那个 KV 几何量,就是最能预测决策伤害的 → **定为 $\Delta$**。起步用**逐列 Pearson/Spearman + 散点图**就够看出结论;进阶可做**多元回归**看哪几列显著、哪些被别的列解释掉,但别一上来上复杂模型。

同时报**欧氏 vs (余弦+norm) 的对照**:拆开赢了(余弦单独 ≈ 欧氏)就丢欧氏只用余弦;打平就直接用欧氏更简。

### 6.4 层归因:哪一层最该刷新/探针该探哪层

两条路拿「哪层致害」:
1. 回归里放**按层分段特征**,看哪段系数最高。
2. 更直接的 **swap-one-layer**:只换 $s_i$ 的第 $l$ 层 KV、其余真值,测归因 KL,逐层得一条「层 → 伤害」曲线。

```
只看 layer 3   : 偏差↔KL 相关 0.2
只看 layer 15  : 偏差↔KL 相关 0.8   ← 中层最致命
只看 layer 30  : 偏差↔KL 相关 0.3
```

这直接告诉你**单层探针该探哪层**:若中层最致命,$\Delta$ 探针就探中层而非图便宜探第一层(接 Exp F)。

## §7 关键细节与陷阱清单

1. **存 logits 要存整个词表分布,不是 argmax** —— 否则 KL 算不了。Step 1 最易埋的坑。
2. **决策 KL 在 unembedding 之后的词表分布上算**,不在最后一层 hidden state 上(§4.3)。
3. **RoPE 对齐先在纯前缀上验证偏差≈0**(§Step 2),否则全是位置伪影。
4. **GQA 用 KV 头数**,不是 Q 头数。
5. **决策 token 锁定到 continue/handoff 那一位**,不对整个 JSON 算。
6. **回归用事件级、避免伪重复**(§6.2);特征用 mean + max + 分层段。
7. **欧氏当对照基线**,大概率被淘汰但必须在场证明拆分有无必要。
8. **多 seed / 多题做显著性**,别单题下结论(同证伪 head 级的严格性)。
9. **锁同模型**(Llama-3.1-8B);跨模型对齐是后话。
10. **M 顺手采集**:本实验已暴露 attention,把 $M_i^{(t)}$ 一起 dump,供 Phase 1 用。

## §8 产出与成功/失败判据

**产出**:
- 一个数据选出的 $\Delta$ 定义(用哪个 K/V 几何量、哪些层),交给 Phase 2 的 gate。
- 漂移结构图(逐 token / 逐 layer / 哪层最致害)。
- 欧氏 vs 拆分的对照结论。

**判据**:
- 某量(如 V 余弦,或中层 K norm,或别的)与决策 KL **强相关** → 它就是 $\Delta$;若 ≠ RelayCaching 的 V 余弦,你得到一个**反 RelayCaching 的实证发现**,本身是贡献。
- **没有任何单一量与决策 KL 相关**(偏差弥散、不可预测)→ 麻烦信号:gate 信号不存在,$\Delta$ 这条路要重想(可能退到「绕开 $\Delta$、直接预测决策翻转」的 plan B)。
- 多数复用事件的决策 KL **本身就极小**(复用几乎不扰动决策)→ 这其实预告了 Exp B 的结论:质量问题可能不存在,贡献会偏向纯成本侧。Exp A 先把这个信号看在眼里,Exp B 再正式定。

## §9 与后续实验的衔接

- **→ Exp B**:Exp A 用 teacher forcing 锁轨迹测「单次复用 KV→决策」;Exp B 放开 teacher forcing,让 reuse 自由跑,测「决策翻转→轨迹分叉→报告差」。Step 0 的测量框架两者共用。
- **→ Phase 1(M)**:本实验顺手 dump 的 $M_i^{(t)}$ 直接进 Exp D(M 稀疏性 + M×Δ 散点);层归因结论进 Exp E/F。
- **→ Phase 2($\Delta$ 落地)**:本实验定的 $\Delta$ 定义,是 Exp G(M×Δ gating 消融)和 Exp H(部分重算)的输入。

> **一句话流程**:full prefill 建金标准轨迹并存全档(含决策完整词表 logits + attention)→ **teacher forcing 把 reuse 钉死在轨迹上(同 token、只换 KV 来源)** → 逐层算 K/V 几何偏差(余弦/norm/欧氏 6 条)→ 算注意力输出偏差(第 7 条,半程)→ 算决策 logits KL(金标准,unembedding 后词表分布)→ **swap-one-summary + 事件级回归**找出哪个 KV 几何量最能预测 KL → 定为 $\Delta$。
