## 多指标看下来的发现
1. **全复用在所有指标上都最差,方向高度一致**:full_reuse vs baseline——F1 22.6 vs 32.7、Prec 23.6 vs 34.3、Rec 28.5 vs 35.4、**全对率 8.0% vs 16.1%(腰斩)**、全错率 57.5% vs 51.7%。六个指标无一例外,不是单一指标的偶然。
2. **"全对率"把伤害放得最大**:最严的"整题全答对且不多答"指标,全复用从 16.1% 掉到 **8.0%(正好一半)**;而 75% ΔK token 拉回 **18.4%(超 baseline)**。说明 KV 复用最容易把"本来全对的题"做坏。
3. **复用伤精确率 > 伤召回率**:full_reuse 的 Prec 掉 **10.7** 分(34.3→23.6)、Rec 只掉 **6.9** 分(35.4→28.5)。复用更多是**多给错答案/答偏**(精确率塌),而非漏答。
4. **75% 重算能救回、ΔK/ΔV-layer 最强**:75% 档普遍回到 0.28–0.33;**8 号 ΔK token(F1 32.8、全对 18.4 全场最高)、7 号 ΔV layer(F1 32.8)、9 号 ΔK layer(召回 37.7 全场最高)三家基本追平 baseline**。只有 6 号 ΔV token 偏弱(28.0)。
5. **75% 明显强于 25%**:25% 档预算太紧,只有 14 号 ΔV token(29.1)还行,其余基本在 24–28 徘徊,信号优势被噪声吃掉。

## "是不是选得准":信号 vs random(同档同粒度,配对 ΔF1)
**这是判断"选择性重算到底靠信号还是靠重算量"的关键。** ⚠ 别再用"random vs full_reuse 不显著"来论证——87 题下 **75% random token 已经显著好于全复用**(+0.054),光重算 75% 的随机位也能救回一截。所以要直接比"信号 − 同档同粒度的 random"。
| 档 | 粒度 | 信号 − random | ΔF1 | 95% CI | 信号更准? |
|---|---|---|---|---|---|
| 75% | **layer** | ΔV − random | **+0.052** | [+0.006, +0.101] | **✓** |
| 75% | **layer** | ΔK − random | **+0.046** | [+0.011, +0.084] | **✓** |
| 75% | token | ΔK − random | +0.048 | [−0.006, +0.108] | ✗(接近) |
| 75% | token | ΔV − random | −0.000 | [−0.043, +0.042] | ✗ |
| 25% | token | ΔV − random | +0.030 | [−0.014, +0.081] | ✗ |
| 25% | token | ΔK − random | −0.016 | [−0.045, +0.011] | ✗ |
| 25% | layer | ΔV − random | −0.039 | [−0.111, +0.035] | ✗ |
| 25% | layer | ΔK − random | −0.038 | [−0.099, +0.021] | ✗ |

**三句话**:
- **"选得准"的硬证据集中在 75% layer**:ΔV、ΔK 两个信号都**显著超过同档同粒度的 random**(CI 不含 0)→ 这一档的恢复确实来自"选对了层",不是单纯多重算。
- **75% token 上信号优势勉强**:ΔK 比 random 高 +0.048 但 CI 擦边(含 0),ΔV 干脆和 random 持平 → token 粒度上"选哪个 token"的信号没立住,主要靠重算量。
- **25% 全线塌**:预算太紧时信号一律打不过 random(几个还反向),噪声完全盖过信号。

## 零成本诊断:ΔK 与 ΔV 在 token 层高度重合(检查2)
每题 `controls/diag.json` 记了 `dk_dv_corr_per_token` —— `dK.sum(0)` 与 `dV.sum(0)`(各自按层求和、逐 token)的相关系数。**这把尺和 token 分支的实际选位口径完全一致**(`select_token` 就是按 `sig.sum(dim=0)` 排 top-X%),所以它直接回答"ΔK-token 和 ΔV-token 选不选同一批 token"。
| 量 | 87 题结果 |
|---|---|
| 均值 / 中位数 | **0.836 / 0.839** |
| 范围 / 标准差 | [0.731, 0.901] / 0.035(极稳) |
| corr ≥ 0.7 | **87/87(100%)** |
| corr ≥ 0.8 | 75/87(86%) |
| `M_rowsum_mean` | 全 0 → 证实 M 不可用、M×ΔV(4/5/12/13)正确跳过 |

**含义(给"ΔK 强于 ΔV"降温)**:token 粒度上两个信号逐 token 排序相关 **0.84、且每题都如此**——它俩基本在选**同一批** token。所以 ΔK-token(32.8)比 ΔV-token(28.0)高那 ~4.8 分,**不可能来自"选了很不一样的 token"**,更像是那 ~16% 不重合尾巴 + run 噪声(况且 ΔV-token vs full_reuse 本就不显著)。**"信号确实有用"的干净证据应落在 layer 粒度**(ΔV/ΔK 都显著超 random),而非 token 粒度的 ΔK vs ΔV 之争。
> ✅ 原盲点已补:diag 没直接记 layer 层相关性,但我用**两个 layer 分支实际选中的层做 Jaccard 重合**反推出来了——ΔV-layer vs ΔK-layer 每题重合 **0.956**(见下节"为什么 layer 信号更好"),即 layer 粒度上两信号也几乎选同一批层、且比 token 层(0.84)更钉死。建议下轮仍把 `dk_dv_corr_per_layer` 正式写进 diag。

## 检查3(layer 弥散污染):代码没实现 + 机理侧这轮算不了,但结论已能从分数反推
原设计的检查3 =「查各层信号求和是否与**注意力弥散度**强相关;若 layer 维各信号 ≈ random layer → layer 选择无效(类比 Exp A head 无效)→ 可砍 layer 分支」。**两点说明**:
- **代码没真落**:`select_and_write` 注释承诺了"检查3 layer 弥散",但 diag 实际只写了 4 个字段(无弥散项)——检查3 未实现。
- **机理版这轮做不了**:它需要注意力权重算弥散度,而本轮 `compute_M` 的 M 全 0(`M_rowsum_mean=0`,与 M×ΔV 同一个卡点:dump 没抓到 report 位注意力)。要补得另跑一轮带注意力的 dump。

**但这个检查要拍的板(该不该砍 layer 分支),结果侧已直接给出答案**(见上"信号 vs random"):
| 档 | layer 上 信号 − random | 判 |
|---|---|---|
| **75%** | ΔV +0.052 [+0.006,+0.101]、ΔK +0.046 [+0.011,+0.084] | 都**显著超** random → **layer 选择有效、不该砍**(恰恰是 75% 最干净的信号) |
| **25%** | ΔV −0.039、ΔK −0.038 | 反而略**低于** random → 这一档 layer 选择**塌成随机**(信号被预算/噪声淹没) |

→「砍 layer 分支」假设在 **75% 档不成立、在 25% 档成立**。机理侧(弥散度污染的因果)需补一轮带注意力权重的 dump 才能验。

## 为什么 layer 信号更好(选层结构分析)
> 先说精确:"layer 更好"**不是**"layer 分数一定更高"(ΔV 是 layer 32.8 > token 28.0,但 ΔK 反而 token 32.8 ≈ layer 32.2)。准确含义是——**"按信号选 vs 随机选"的优势在 layer 档干净显著、在 token 档飘**(见上"信号 vs random")。下面拆为什么。

**决定性证据:layer 选择几乎是一条"固定动作"**。把 87 题实际选中的层(`controls/branch_07/09.json` 的 `recompute_layers`)扒出来:

| | ΔV-layer(b7) | ΔK-layer(b9) |
|---|---|---|
| 每题选 48/64 层重算 | ✓ | ✓ |
| **恒被选**(100%题都重算)的层 | 47 层:**第 17–63** | 46 层:**第 18–63** |
| **从不被选**(留着复用)的层 | 第 **0–15**(浅层) | 第 0–14(浅层) |
| **ΔV vs ΔK 选同一批层?**(每题 Jaccard) | **0.956**(随机对照仅 0.60) | |

读法:这个"信号"几乎每题都在干同一件事——**"前 ~15 层浅层留着复用、第 16–63 深层全部重算"**;而且 ΔK/ΔV 两把尺在 layer 上高度一致(0.956,token 层只有 0.84)。

**为什么这就打得过 random** —— 因为 **KV 复用的伤害是「按层」分布的,不是均匀撒在每个 token 上**:数据显示浅层(0–15)复用版 vs 真算版 K/V 几乎不漂 → 留着复用没事;深层(16–63)漂得多 → 必须重算(与既往 KVCOMM 结论"该重算的是中后层、sink/浅层安全"一致)。于是信号稳稳"保住深层、放过浅层",每题都对;random 随机丢 16 层,运气差就丢掉关键深层(伤质量)+ 白算浅层(浪费)。

**为什么 token 那档立不住** —— 两个原因叠加:
1. **统计上 token 打分天生噪**:每题摘要 token 中位 ~2700(最多 1.5 万),给几千个带噪的 per-token 漂移分排序、挑前 75%,切割线附近全是掷硬币;而 layer 只在 64 个里挑,每层分还是**对几千 token 求和**得来——求和把 per-token 噪声平均掉,排序又稳又准。这正解释了 ΔK/ΔV 重合度 token 层 0.84(飘)、layer 层 0.956(钉死)。
2. **结构上 token 跟"按层的伤害"拧着**:选中一个 token 是在**所有 64 层**重算(含本可省的浅层=浪费),没选中的在所有层复用(含危险深层=该算没算)。**token 粒度根本表达不了"跳过浅层"这个最关键的省法**,等于逆着伤害的形状使劲。

**诚实边界**:
- layer 的"信号"其实退化成一条**近固定规则**"前 15 层复用、后面重算"(几乎不随题变)→ 严格说一条**手写死的"跳过前 15 层"**就能拿到差不多效果;layer 信号的价值更多是"验证并锁定了这条结构先验",而非每题动态选得多聪明。
- 实用副产品:**只重算后 75% 的层(第 16–63),质量基本追平 baseline(32.8/32.2 vs 32.7)→ 前 15 层浅层可近乎免费复用、省 ~23% 层算量**。
- ΔK-token 也摸到 32.8(不比 layer 低),只是统计上飘;所以是"layer 稳 / token 看运气+具体信号",不是 token 一无是处。

## 怎么读 / 注意(诚实边界)
- **ΔK 在 75% 档数字最好(尤其 token,全对率 18.4 超 baseline),ΔV 更吃 layer 粒度;但 token 粒度上 ΔK/ΔV 选位相关 0.84(见上节诊断),"ΔK 优于 ΔV"在 token 档站不住,别过度解读** —— 与原假设(主用 ΔV)不同,待补 M 后(M×ΔV 主方案)再一并核。
- **绝对量级中等**:复用伤害(baseline−reuse)0.101、恢复幅度 0.05–0.10,**都没过 0.14 的四臂跨run噪声底**;结论宜表述为"**方向清晰、75% layer 的 ΔV/ΔK 与 ΔK-token、25% ΔV-token 配对显著、但量级中等、25% 不稳**",别夸成"大幅"。
- **样本是 87 题不是 100**:driver 在 q87 途中停了(`q87/` 缺 `reports.json`)。87 题已足够看清方向;若要凑满 100 提高功效,需重启 driver 补 q87–q99 再重打分重聚合(server 仍在跑,PID 见 `server.log`)。
- **本轮仍缺 M×ΔV(主方案)**:无法回答"注意力加权选位是否更优";要等把 M 的注意力前向单独抓出来,再跑分支 4/5/12/13。
