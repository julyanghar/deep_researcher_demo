# 全段实验主线 · 40 题报告(放任复用 vs 全程真算,大白话版)

> **状态**:40/40 题跑完,env2(决策)、env3(内容分叉,免搜版)、env4(报告打分)都算了。
> **一句话(已上修)**:**复用让 agent 研究过程大幅跑偏(轨迹偏离、来源几乎全换),报告答案也变差——换合适统计量后"变差"borderline 站住了(不是当初平均值看的那样"不显著")。**

> ### ⚠ 重要更新(初版"不显著"已被零成本重分析上修)
> 初版主指标是"逐题 F1 差的平均"(Set 0.056、CI 跨 0、判不显著)。但 40 题极不同质(~19 题持平、几题重创、1 题反向),**均值是错的统计量**。换 **符号检验 + 分层 + 分布**(详见 [env4_reanalysis.md](env4_reanalysis.md))后:
> - **全 40 题配对 F1 差 = 0.151 [0.041, 0.272]**(CI 不跨 0,显著)。
> - **符号检验(全 on-topic):变差 10 / 变好 2,p≈0.039 显著**。
> - **分层:过程偏离的题 F1 掉 0.24(显著);没偏离的题≈0(阴性对照)** → "过程被搅乱→答案变差"因果链成立。
> - 仍 borderline(只在合并 on-topic 显著、p 刚过 0.05、阈值 0.01 不显著)。
>
> **两轨 deepsearchqa 总评分**([metrics/](metrics/),与官方 metrics.json 同口径,全 40 题):
> | | precision | recall | **F1** | 全对率 | 完全错率 |
> |---|---|---|---|---|---|
> | 真算 prefill | 39.5% | 41.0% | **37.0%** | 20%(8/40)| 45%(18/40)|
> | 放任 reuse | 26.1% | 29.7% | **25.8%** | 12.5%(5/40)| 60%(24/40)|
>
> (这个口径含 off-topic 跑题题,所以差距(F1 −11 分、完全错 +15 分)比 §4.3 的 on-topic 纯质量退化更大。)

---

## 一、想证明什么(大白话)

我们怀疑:**deep research agent 里,把前面算好的 KV 缓存反复拿来复用(省算力),会把后面的研究带偏、最后报告变差。** 要把这条链一环一环证出来:

> 复用 KV →(决策被带偏)→(followup 问题/研究方向跟着偏)→(搜到、用到的资料就不一样了)→ **最终报告答案变差**

复用只在两个地方起作用:**主管(Supervisor,每轮决定"继续还是收尾"+ 提下一步问什么)** 和 **写手(Writer,写最终报告)**。所以实验只盯这两处。

---

## 二、怎么做的(数据从哪来)

每道题**跑两遍**同一个 agent,只差"复不复用":

- **真算轨(prefill)**:每段都老老实实重算,不复用 —— 当**标准答案/基准剧情**。
- **放任复用轨(reuse)**:全程把前面 worker 写的摘要的 KV 拿来复用 —— 误差会**一路累积、轨迹自己越走越偏**。

两遍都记下:**每轮主管的决定(继续/收尾)+ 每轮提的 followup 问题 + 最终报告**。题目用 `google/deepsearchqa` 前 40 道(自带标准答案),模型统一 Qwen3-32B,全程贪心解码(temp=0,理论上同输入同输出)。

**自检**:真算轨跑两遍,**每轮决定完全一样** → 后面看到的"偏离"是复用造成的,不是模型自己抽风。(报告文字两遍会略有不同,是 GPU 浮点的小抖动,所以报告打分用"答案对不对"而非"逐字一样"。)

---

## 三、算了哪几样、为什么(大白话)

整条链分三个"环",分别量:

- **环2 — 决策偏没偏**:把两轨**每轮的"继续/收尾"**并排比,**第一次不一样的轮**就是偏离点。只看决定本身,简单可靠。
- **环3 — 研究内容分没分叉**(这次免搜、只用已记的数据):
  - **followup 问题相似度,怎么算**:把两轨**同一轮**提的所有 followup 问题,各自**拆成词的集合**(去标点、小写),算 **Jaccard = 两边共同的词数 ÷ 两边合起来的词数**(0~1)。1=用词完全一样,0=完全不同。
    > 例(q13 第1轮):真算问"…2010 **Eyjafjalla 火山** … **flooding** event …",复用问"… **architectural records** … **orientation** …",共同词很少 → Jaccard 低 → 判两轨研究方向已经不一样。
  - **怎么判"问题方向偏离了"**:某一轮的 followup Jaccard **< 0.5**(超过一半的词都对不上)就记这轮"内容偏离";一道题**第一次出现这种轮** = 它的"内容偏离轮"。⚠ **0.5 是人为定的阈值**,所以这个判断**依赖 prefill 两遍的基线来校准**(基线若也才 0.5,说明这是噪声不是偏离)——见 caveat。
  - **报告来源,怎么算**:用正则把两份报告正文里所有 `http(s)://…` 网址抠出来,各成一个集合,同样算 Jaccard。低=两轨最后引用的资料几乎不重叠。
- **环4 — 报告答案对不对(最关键)**:拿标准答案给两份报告打分,用大模型评委(qwen-flash)判**答案正确性**,折成 **F1 分**(0~1,1=完美)。
  - **为什么用 F1 不用"全对率"**:F1 是连续分,能看出"答对了 5 项里的 4 项"这种**部分退化**;"全对率"是 0/1,漏一项就整题算 0,看不到细微变差。
  - **"差(真算−复用)"是什么**:每道题真算报告、复用报告各打一个 F1,**真算 F1 − 复用 F1**,再对所有题求平均。**正数=真算更好=复用变差了**,数值就是"复用平均掉了几分"。
  - **"[95% CI]"是什么(置信区间)**:因为只有几十道题、是抽样,这个"差"有不确定性。95% CI = 用 bootstrap(反复重采样)估出来的、**真实的差大概落在哪个范围**。**如果区间跨过 0**(如 `[−0.064, 0.183]`),说明真值**有可能是 0 甚至负** → 这个"变差"**统计上站不住(不显著)**;只有**整段区间都在 0 以上**才算"确实变差了"。
  - **Set 题 / Single 题分开看**:Set 题答案是一组(比如"哪几个国家"),F1 细;Single 题答案是一个,F1 就是对错。混在一起会互相稀释。
  - **先挑出"跑题"的题单列**:复用可能让 agent **跑去答别的问题**。这种拿原题标准答案打分当然低分,但那是"跑题"不是"答得差",得分开。所以另让评委判一下"这报告还在答原题吗"。

## 四、结果(40 题)

### 4.1 环2:决策偏离(只看继续/收尾)
- **15/40 题**两轨决定就不一样了;偏离轮分布 `{第1轮:5, 第2轮:7, 第3轮:3}`。
- 其中 **8 题复用轨更早收尾**(本该多研究却提前停),也有反过来"该停不停"的(如 q3、q15)。

### 4.2 环3:研究内容分叉(这次补上的中间环)
| 信号 | 数值 | 大白话 |
|---|---|---|
| **合并偏离题**(决定 或 问题方向 任一偏)| **27/40** | 比只看决定的 15/40 **多抓 12 题** |
| 合并首次偏离轮 | `{第1轮:19, 第2轮:7, 第3轮:1}` | **多数第 1 轮就开始岔** |
| followup 问题相似度 | 第1轮 0.47 → 第2轮 0.33 → **第3轮 0.28** | **越往后两轨问的越不一样**(偏差累积)|
| **报告来源(URL)相似度** | **平均 0.13** | 两轨最后**用的资料 ~87% 不重叠** |

> 相似度 = 共同词(或共同 URL)÷ 合起来的词(或 URL)= Jaccard;某轮 followup 相似度 **< 0.5** 记"问题方向偏离"。算法 + q13 例子见 §三。

→ **关键发现**:只看"继续/收尾"会**漏掉一大半偏离**。很多题决定一样,但**复用把"下一步问什么"带偏了**(env2 漏的 12 题),研究方向一路走歪、最后用的资料几乎全换。这条机制链是实打实成立的。

### 4.3 环4:报告答案对不对(最关键,但这里偏弱)
| 指标 | 真算 | 复用 | 差(真算−复用)[95%CI] | 站得住吗 |
|---|---|---|---|---|
| **Set 题 F1**(21 题)| 0.431 | 0.375 | **0.056 [−0.064, 0.183]** | ❌ **CI 跨 0,不显著** |
| └ 精确率(precision)| | | 0.086 | 复用更爱**夹带错答案** |
| └ 召回率(recall)| | | 0.020 | 漏正解几乎没变 |
| **Single 题 F1**(10 题)| 0.40 | 0.10 | **0.300 [0.000, 0.600]** | ⚠ 临界(刚好不跨 0)|
| % 完全答对 | | | 0.097 [0.000, 0.226] | ⚠ 临界 |
| 本地字面打分 | | | −0.004 | **不可信**(报告没"## Final Answer"节,退化成全文匹配)|

> **差** = 真算 F1 − 复用 F1(对所有题求平均),正数 = 复用变差。**[95%CI]** = 这个差的真值大概落在哪(bootstrap 估);**跨过 0 = 统计上站不住(可能本来就没差)**。详见 §三。

**老实说**:方向全对(复用都更差),但 **40 题上幅度小、统计没站稳**——Set 题 F1 差的置信区间跨过 0(可能为 0),Single 题刚好临界。20 题时看着强,是小样本的运气。

### 4.4 复用让 15% 的题彻底跑题(这个是稳的)
评委判"还在答原题吗",并**对比真算轨**(只有"真算在题、复用跑题"才算复用害的):
- **干净归因给复用的跑题 = 6/40(15%)**:q2、q7、q9、q11、q23、q35。
- 两轨都跑题(题本身难、不怪复用)= 3 题:q4、q16、q25。

### 4.5 危害到底从哪条路来(把三个环对起来看)
按"哪些题报告变差了(F1 掉 >0.05)"分通道:

| 通道 | 题(F1 掉多少) | 说明 |
|---|---|---|
| **① 决定就变了** | q3(1.0)、q9(1.0)、q29(1.0)、q32(0.8)、q15(0.1)、q11(0.07) | 继续/收尾序列变了 → 报告差 |
| **② 决定没变、但问题方向被带偏**(env2 会漏)| q13(1.0)、q23(0.8)、q22(0.67)、q20(0.19)、q37(0.17) | 主管"问什么"被复用带歪 → 研究内容分叉 → 报告差 |
| **③ 都没偏、报告却差** | q6(0.06)、q8(0.1) | 幅度很小,大概率是 GPU 浮点噪声 |
| 复用反而更好 | q27(−0.8)、q38(−0.09) | 2 题(q27 还差挺多)|
| 持平 | 25 题 | 多数没变 |

**具体长啥样(q13)**:问"冰岛火山致航空中断那年…某地最大降雨…那地城堡朝哪个方向?"标准答案=**西**。
- 真算轨:正确追到 **Exeter Castle(2012 St. Jude 风暴)→ 朝西** ✓
- 复用轨:决定**一模一样**,但 followup 没再锚定年份、泛泛问朝向 → 追到**错城堡(Castlerigg)→ 朝东** ✗
→ 报告 5057 字、写得满满当当,但**结论错**。这就是"决定没变、问题方向被带偏"的典型。

## 五、怎么解读(诚实)

- **机制层(环2+环3)很强**:复用让 **27/40 题轨迹偏离**、**研究方向越走越偏**、**最终来源 ~87% 不同**、**15% 直接跑题**。"复用把研究过程带歪"这件事,证据扎实。
- **结果层(环4)borderline 站住**(已上修,见顶部更新 + [env4_reanalysis.md](env4_reanalysis.md)):换符号检验后**全 on-topic 变差 10/变好 2(p≈0.039)**,分层显示**过程偏离题 F1 掉 0.24(显著)、未偏离题≈0(阴性对照)**。初版"不显著"是被"平均值 + 拆分 + 持平稀释"坑了。仍 borderline(合并才显著、p 刚过 0.05)。
- **合起来**:**复用把过程搅乱了,而且"搅乱"确实倾向翻译成"答案变差"**(尤其过程偏离的题);所有重创题都落在偏离层。**thesis 方向成立、机制清楚、定量 borderline 站住**,加样本即可坐实。

## 六、必须诚实的 caveat

1. **环4 仍 borderline**(已从"不显著"上修,见 [env4_reanalysis.md](env4_reanalysis.md)):全 on-topic 符号检验 p≈0.039,但**只在合并时显著、Set/Single 单拆不显著、阈值 0.01 不显著**。要变铁证仍需**加样本**(60~100 题)。
2. **环3 还差一个基线**:followup/URL 的低相似度,得用 **prefill 跑两遍**当基线证明"不是浮点噪声本身就这么低"。**没跑这个基线前,env3 的数只能算"强烈提示"、不算铁证。** 尤其 round1 的 0.47 卡在阈值附近,最依赖基线。
3. **本地字面打分失效**:writer 没输出 `## Final Answer` 节 → 退化全文匹配 → 那个"≈0"是假象,**只看大模型评委**。
4. **评委本身有噪声**(已用多判取多数缓解,但 off-topic 判定仍会偶尔翻)。
5. **env3 是免搜代理版**(followup 文本 + 报告 URL),不是"实际检索的 URL"。最忠实的版本要重跑、把每轮 query+URL 记进轨迹(留给探针轨阶段一起做)。

---

## 七、结论 + 下一步

**结论**:复用对研究**过程**的破坏证据很强(偏离 27/40、来源 87% 不同、跑题 15%);对**结果**(报告答案)的破坏**borderline 站住**(符号检验 p≈0.039、过程偏离层 F1 −0.24 显著 + 阴性对照,见 [env4_reanalysis.md](env4_reanalysis.md))。所以"复用有害"目前是**机制清楚、定量 borderline 成立、加样本即坐实**。

**下一步**:
1. **跑 env3 确定性基线**(prefill 两遍)→ 把 env3 从"提示"升成"铁证"。
2. **加样本到 ~60–100 题** → 让 env4 的 F1 差站稳(或确认真的小)。
3. 视情给 writer 加 `## Final Answer` 节,修本地旁证。
4. (最忠实版 env3)重跑时把每轮 **researcher query + 返回 URL** 记进轨迹。
5. 主线立住后 → 解法验证(探针轨 |Δz| + M×Δ 选择性刷新)。
