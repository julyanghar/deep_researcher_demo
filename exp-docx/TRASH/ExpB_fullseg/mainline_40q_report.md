# 全段实验 · 40 题终版报告(四臂分解 + 噪声底,真·KV 复用)

> **状态**:四臂(A_orig / A_orig2 / B / C)各 40/40 跑完;env2/3(过程偏离)、env4(报告 F1,配对 + bootstrap CI)、四臂 metrics.json(qwen-flash 评委,打在真实报告上)全算完。
>
> **一句话结论**:**纯 KV 复用对逐题 on-topic 答案质量没有统计显著的危害(配对 Set F1 差 0.045,CI 跨 0、且低于噪声底);分隔符也无害(0.000)。但复用轨整体 F1 比真算低 ~6-8 分,源头是少数(~3 题)被复用带跑题/答错的"崩题",而非系统性退化。在 40 题、噪声底巨大的前提下,这点整体差距"方向上像真害、统计上不airtight"。**

> ### ⚠ 后续 kimi-k2.5 复核**推翻了"复用整体 F1 低 8 分"**(见 §八)
> 下面 §4.2/4.3 的 F1/metrics 全用 **qwen-flash** 评委。后查出 qwen-flash **自漂移极大**(同批报告两次打分逐题 |Δ|=0.236)。换更强更稳的 **kimi-k2.5**(自漂移仅 0.018、约稳 13 倍)复核同一批报告:**复用臂不再低于真算、甚至略高**(B 24.8% < SW 30.3% / C 28.2%),supervisor/writer 复用效应均 ≈0。→ "C 低 8 分 + 3 崩题"**主要是 qwen-flash 评委噪声的假象**,不是 KV 复用的真实危害。**§四/§五 的整体差距结论以 §八 为准。**

> ### ⚠ 这版**推翻**了旧版"复用有害 borderline 站住"
> 旧版(本文件历史版本)的"复用轨"其实是 **control-truth bug**(`LMCACHE_BLEND_CONTROL` 残留 → 每个 blend 被强制全重算,**根本没复用 KV**,见 [claude-docx/14](../../claude-docx/14-blend-control-truth-pitfall.md)),且**没有噪声底**。这版用**真 CacheBlend 复用**(已验 reuse~99%)+ **第二条 prefill 当噪声底**,结论反转:之前看到的"大幅偏离 + 答案变差"**绝大部分是 agent 非确定性**(两条相同 prefill 就有 38% 决策偏离),不是 KV 复用。

## 一、想干净回答的问题

旧实验把"分隔符效应"和"KV 复用效应"混在一起,且复用轨有 bug。这版**拆开 + 加噪声底**:

> **总效应(C−A)= 分隔符效应(B−A) + 纯 KV 复用效应(C−B)**

- 复用轨为切段必须用特殊分隔符 `<|fim_pad|>`(prefill 用 `\n\n`)→ 先单独量"分隔符"本身的影响。
- 再量"在带分隔符基础上,KV 到底复用没复用"的纯效应。
- 关键:agent **非确定性极大**(temp=0 也漂),必须有**噪声底**(两条相同 prefill 互比)来判"效应是否超过本底"。

## 二、四臂(各 40 题,google/deepsearchqa,Qwen3-32B/TP4)

| 臂 | 含义 | prompt | KV | 角色 |
|---|---|---|---|---|
| **A_orig** | 真算·无分隔符 | `\n\n` | 无复用 | prefill 基线(早先跑)|
| **A_orig2** | 真算·无分隔符(第二条)| `\n\n` | 无复用 | **噪声底伙伴**(本轮重跑)|
| **B** | 真算·带分隔符(vanilla 全重算)| `<\|fim_pad\|>` | 无复用 | 隔离分隔符 |
| **C** | 复用·带分隔符(CacheBlend)| `<\|fim_pad\|>` | 复用~99% | 纯 KV 复用 |

- **噪声底 = A_orig vs A_orig2**(两条配置相同的 prefill,差异 = 纯 vllm 非确定性 + 搜索漂移)。
- **分隔符效应 = B − A_orig2**、**纯复用效应 = C − B**、**总 = C − A_orig2**(B/C/A_orig2 都在本轮同条件 replay 跑,最干净)。
- 运行细节(命令、env、验证):见 [experiment_workflow.md](experiment_workflow.md);C 确认走 CacheBlend(`BLEND_PATH=CacheBlend` reuse 99%/0 CONTROL),B 确认纯 prefill(0 BLEND_PATH + KVTransfer disabled),A_orig2 确认无分隔符(sep=off)。搜索 replay 命中 ~60-82%。

## 三、方法(算了啥、怎么算)

- **env2 决策偏离**:两臂每轮"继续/收尾"序列,第一次不同的轮 = 偏离点。
- **env3 内容分叉**(免搜):同轮 followup 问题文本 Jaccard + 报告正文 URL 集合 Jaccard(越低越分叉)。
- **env4 报告 F1(命脉)**:评委 **qwen-flash**(闭源,DashScope)判答案正确性 → P/R/F1。**配对**逐题算 `基线−处理`,**bootstrap 95% CI**;on-topic 分层(评委判报告是否还在答原题);Set/Single 分开。
- **metrics.json**:每臂 40 份报告各自跑 deepsearchqa 官方口径(`aggregate_ratings`)→ 整体 precision/recall/f1 + 全对率(带 CI)+ by_category。和 `eval/results/RUN_real50_normal` 同口径。
- **判据**:某效应的 CI 不跨 0 **且** 幅度超过噪声底 → 才算真效应。

## 四、结果

### 4.1 env2/3:过程偏离 vs 噪声底(全 40 题)
| 对比 | 决策偏离(status) | 报告 URL-Jac(↑=越像) | 合并偏离 |
|---|---|---|---|
| **噪声底** A_orig vs A_orig2 | **15/40 (38%)** | **0.15** | 23/40 |
| 分隔符 B vs A_orig2 | 20/40 (50%) | 0.21 | 27/40 |
| 纯复用 C vs B | 17/40 (42%) | 0.24 | 24/40 |
| 总 C vs A_orig2 | 19/40 (48%) | 0.20 | 28/40 |

→ **噪声底巨大**:两条**完全相同配置**的 prefill 就有 38% 决策偏离、URL-Jac 0.15。**纯复用(42%/0.24)基本贴着噪声底**(URL-Jac 0.24 甚至比噪声底更像);分隔符(50%)略高于噪声。**旧版"复用让 27/40 跑偏"的轨迹分叉,绝大部分是非确定性,不是复用。**

### 4.2 env4:配对报告 F1 差 + bootstrap CI(qwen-flash)— **核心**
Set 题(主力,细粒度):
| 对比 | Set F1 差 | 95% CI | 判读 |
|---|---|---|---|
| **噪声底** A_orig vs A_orig2 | **0.142** | **[0.008, 0.291]** | 地板就 0.14(含 A_orig 旧搜索 confound)|
| **分隔符** B − A_orig2 | **0.000** | [−0.112, 0.121] | **无效应** |
| **纯复用** C − B | **0.045** | [−0.052, 0.161] | **不显著,且 < 噪声底** |
| 总 C − A_orig2 | 0.025 | [−0.071, 0.127] | 不显著 |

Single 题:噪声底 0.000、分隔符 −0.111、纯复用 0.074、总 0.147 —— 全部 CI 跨 0,不显著。

→ **逐题 on-topic 答案质量:分隔符 0、纯复用 0.045(CI 跨 0 且低于噪声底 0.142)→ 没有统计显著的危害。**

| 臂 | F1 | precision | recall | 全对率 |
|---|---|---|---|---|
| A_orig | 32.33% | 34.22% | 36.45% | 17.5% (7/40) |
| A_orig2 | 31.74% | 31.27% | 34.07% | 20.0% (8/40) |

### 4.4 C 整体为何偏低:深挖(解决"配对不显著 vs 聚合低"的张力)
配对说"纯复用不显著"(0.045),聚合说"C 低 8 分",矛盾点在 **C 的整体差距集中在极少数"崩题"**:
- **C 比 B 差 ≥0.5 的题只有 3 道**:**q27**(Set,B=1.0→C=0,在题但答错)、**q34**(Single,1.0→0,**复用致跑题**)、**q5**(Single,1.0→0,跑题)。
- **干净归因给复用的跑题 = 3/40**:q34、q2、q30(prefill 在题、reuse 跑题)。
- 这 3 道崩题(都从 1.0 掉到 0)把聚合 F1 拉低 ~6-8 分;但因为只占 3/40、逐题方差大,**配对均值差(0.045/0.074)的 CI 跨 0**。

→ **机制**:纯 KV 复用偶发地把个别题带跑题/答错(~3/40),不是普遍的"答案逐题变差"。

## 五、结论(诚实,大白话)

1. **分隔符无害**:B(带分隔符全算)≈ A_orig/A_orig2(无分隔符全算),配对 0.000、聚合 34% vs 32%。旧版担心的"分隔符 + warmup"对答案没影响。
2. **纯 KV 复用:逐题 on-topic 答案质量无显著危害**(配对 Set 0.045、Single 0.074,CI 都跨 0,且 < 噪声底 0.142)。
3. **但复用轨整体 F1 低 ~6-8 分(26% vs 34%)**,源头是 **~3 道"崩题"**(被复用带跑题/答错,从满分掉到 0),其中 3/40 干净归因于复用致跑题。**整体层面 C 离群于三条全重算臂(它们只差 2 分)→ 方向上像真害。**
4. **最大的发现是噪声底巨大**:两条相同 prefill 就有 38% 决策偏离、配对 Set F1 差 0.142。**旧版"复用大幅有害"绝大部分是非确定性 + control-truth bug + 缺噪声底造成的 artifact。** 真复用的危害比旧版小得多,且没在逐题层面站稳。

**综合判定**:**纯 KV 复用不会系统性地让答案变差;它偶发地(~3/40)把个别题带跑题,拖低 micro 整体 ~6-8 分。** 但这"~3 道崩题"**和噪声底的逐题摆动分不开**:§4.5 逐题表显示,两条**完全相同配置**的 prefill(A_orig vs A_orig2)同样有 q9/q29/q32 等 1.0→0 的崩题,且 **|C−B 宏差 0.07| < |噪声底宏差 0.08|**。所以:**micro 聚合看 C 像离群(−8 分),但逐题/宏平均看,复用的害落在非确定性包络内、统计上不可分。要坐实"复用偶发带跑题"是真效应还是噪声波动,需加样本(~80-100 题)。**

## 六、必须诚实的 caveat

1. **噪声底有 confound**:A_orig 是早先跑的(搜索条件/时间不同),所以 A_orig vs A_orig2 的 0.142 偏大、含搜索漂移。但**干净的分隔符/纯复用效应(B/C/A_orig2 同条件 replay)= 0.000 / 0.045,本身就很小**,不依赖这个偏大的噪声底也成立。
2. **配对 vs 聚合的张力**:配对(on-topic、逐题)说不显著;聚合(全题、micro)说 C 低 8 分。两者都对——前者是"逐题质量",后者被少数崩题主导。本报告两个都报。
3. **n=40 + 效应集中在 ~3 题** → 配对统计力不足;加到 ~80-100 题才能判"复用偶发带跑题"是否稳。
4. **搜索只 held ~60-82%**(replay 命中率;query 计划漂移导致冷 miss 走 live)→ B/C 间残留搜索差异,计入噪声。
5. **评委(qwen-flash)有噪声**;off-topic 判定偶尔翻。

## 七、下一步
1. 加样本到 ~80-100 题,坐实"复用偶发带跑题"(~3/40)是真效应还是噪声。
2. 看那 3 道崩题(q27/q34/q5)的轨迹,确认是不是复用把研究带歪。
3. **解法验证(探针轨)**:单点 |Δz| + M×Δ 选择性刷新——绕开非确定性,直接测"哪些 token 驱动决策、该刷新谁"(plan 已写,见 [callchain_cdriver_and_control.md §6](callchain_cdriver_and_control.md))。

## 八、评委复核(kimi-k2.5)— 推翻"复用整体更差",并验证评委可信
§4.2/4.3 全用 qwen-flash。后续发现它评委**自漂移极大**,改用更强的 **kimi-k2.5** 统一复核:每臂逐题各打**一遍**(不再走 merge 把 SW 重复打)、逐题落盘、官方 `aggregate_ratings`。脚本 [exp_sw_score.py](../../server/vllm/Exp_sw/exp_sw_score.py)、[exp_sw_stability.py](../../server/vllm/Exp_sw/exp_sw_stability.py);数据 `sw_unified/`。新增 **SW 臂**(supervisor 复用 + writer 纯 prefill,已验证)隔离 supervisor 单独的影响。

**① 整体 F1 反转**(同一批报告,换评委):
| 臂 | qwen-flash | kimi-k2.5 |
|---|---|---|
| B(真算·基线)| 33.92%(最高)| **24.82%(最低)** |
| C(全复用)| 26.10%(最低)| **28.18%** |
| SW(只 supervisor 复用)| — | 30.28% |
| A_orig / A_orig2 | 32.3 / 31.7 | 31.1 / 30.5 |

→ 两评委把 B vs C 的高低**判反**。kimi 下复用臂(SW/C)**≥** 真算 B。

**② 评委自一致性**(同批报告打两遍,mean|Δf1|):qwen-flash **0.236** vs kimi **0.018**(稳约 13 倍)。→ kimi 可信;qwen-flash 的"C 低 8 分"是其评委噪声,非复用真害。

**③ 配对效应**(kimi,Set,run1/run2 两次):supervisor B−SW = −0.007 / +0.024、writer SW−C = +0.006 / −0.018、总 B−C = 0.000 / +0.006 —— 全 **≈0、贴 0 抖、在噪声底内**。
