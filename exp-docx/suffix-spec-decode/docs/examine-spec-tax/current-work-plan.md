## 〇、创新主线(用户定调,7/5 夜):选择性投机解码

前提两条已坐实:①Ceager 把可消的固定开销砍完(大病=驱动锁等待 4.5 已消),剩余账单以**按请求可躲的按量费**为主;②summary 投机效果双峰(0.81x~1.76x)——**全开与全关都不是最优,按请求选才是**。经济学边界:并发下每步机件成本是批级共享、关单个请求省不掉;**选择性投机的收益 = 输家们的里程费(ctx×草稿)+白验证费**,恰好是双峰里输家亏损的主因。

**三层门控设计**(全部实现于 suffix proposer 内部,返回空草稿即"关",不动调度/API):
1. **静态门**(请求开始前,零成本):tag=report 永远放行;tag=summary 且 prompt>X → 降 k/不押(依据:里程费斜线+逐条"长 prompt 必亏");
2. **动态门**:前 N 个有草稿步全拒 → 关(A3 的 N-净省秒曲线定 N);
3. **探针重开**:关后每 M 步试探 1 个草稿(依据:热树迟到爆发,一刀切会错过兄弟输出入树;A3 翘头占比定去留与 M)。

参数来源:A2→X 与题型先验;A3→N、M、策略形态;A1→收益上限预算(输家亏损总量=可回收池)。

**单 server 可行性(已确认)**:按请求关投机可行——proposer 内策略对该请求返回空草稿即可(C0 的逐请求版,零 API/调度改动);但**执行模式是引擎级**,同 server 做不到"某些请求走纯图 A"。被门控请求 = "Ceager 减草稿"。**C0eager−A 的 +4.5 按"谁付账"分类(部署问题的真正答案)**:server 级会员费 4.3(发射件数×10 得 1.7 + cond 等待 1.2 + 零草稿仍走 spec 采样路径 0.9[待C0piece] + 异步关 0.5,批里有人投机就反正要付)+ **逐请求仅 0.2**(propose 空跑,门控入口短路可压到 ~0.05)。故:混合批边际≈原生 → 单 server 成立;独占批扛全额 28.5(慢 19%)是下界。**候选微优化**:"整批无草稿的步退回普通采样路径"(几十行,把全员门控地板 28.5→~27.6)。C0eager 一格双答:税表终审(科学)+门控残余成本(部署)。

## 二、分析计划(预注册,判据先写死;数据落地后执行)

### 自检三关(任何分析前先过)
- [ ] token 守恒:每请求轨迹 Σ(acc+1) ≈ llm_calls 的 completion 步数记录(差>2 = 对齐 bug);
- [ ] 草稿守恒:全体 Σdr ≈ /metrics 的 draft_tokens_total 快照;
- [ ] 时钟单调:每 pid 轨迹文件内 t 严格递增;同 t 行数 = 该步批大小,分布应合 MAX_CONCURRENCY=3。

### A1 最优配置端到端加速表(Ceager vs A,同天 40 题)
- summary/report 分开:聚合 tok/s 加速比 + 分桶普查(tag×prompt 长度桶,>1.05x 加速/0.95-1.05 平/<0.95 亏 三档比例);
- **逐请求命中率分布**(轨迹 Σacc/Σdr,req_id join llm_calls 拿 tag/题号)——首次落到单请求粒度。

### A2 —— **已判(7/6 晨)**:q17/q1 亏损=agent 路径方差(decode 层 0/40 亏);题型效应方向存在幅度有限(20.3~34.0%,cov12 ρ=0.374,可得性/长度 ρ≈0);**D 系列收敛为机制侧抬升**(长匹配/打分/结构化引导),分题型调度收益有限

候选信号(按可用时点):①tag(report 放行,已知最强免费信号) ②prompt 长度(冷树 ρ=-0.56,待热树复核) ③兄弟互抄可得性 ④首步草稿长度 dr₀ ⑤头 N 步连续 miss(用户原始假设) ⑥滑动命中率(重开用)。
