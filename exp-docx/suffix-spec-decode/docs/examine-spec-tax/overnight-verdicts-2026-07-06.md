## 三、机制判决:税表终版(八修收口)

### spec×执行模式 2×2(SUMMARY 每步 ms,终局)

| | spec 关 | spec 开(空草稿) |
|---|--:|--:|
| graph/混合 | A = 24.0 | C0 = 34.0(+10.0) |
| 全 eager | F = 24.5 | **C0eager = 26.4(+1.9)** |

### 税的三段式(最终归因)

```
suffix(Ceager 形态)每步 33.9 = F 基线 24.5
  + spec 管线 1.9   ← 执行模式无关(propose/RPC 0.2 + 异步关 0.5 + memcpy 0.5 + 零头)
                      几乎不可治也不值得治;门控请求的"会员费"就是它
  + 草稿费 7.5      ← 新战场:非线性(激活费+按量假说,晨拆);
                      含 sampler/metadata 1.26(纯草稿驱动,C0piece 直测零草稿=0)
suffix(混合形态)另付边界税 8.1 = 驱动锁等待 4.5(cuLaunchKernel 内 cond wait,栈实锤)
  + 图段启动 1.4 + 杂项 ~2.2   ← --enforce-eager 一刀全消
```

### 今晚锁死的底层物证

- **驱动锁等待 4.5ms/步**:发射线程停在 `cuLaunchKernel` 内部驱动私有条件变量(调用栈:`pthread_cond_timedwait ← libcuda ← cuLaunchKernel`);全 eager 对照 **0.00s**——graph/eager 交错提交时驱动内部串行化,"慢发射 37.6μs 重尾"本质是等锁;
- **⑥ sampler/metadata 纯草稿驱动**:C0piece 零草稿下 STEP_TIMING 零调用(路径根本不执行);
- **门控地板**:空草稿请求在 Ceager server 上独占 = 26.4(仅慢 A 10%),混合批中边际 ≈0——单 server 选择性投机的成本模型成立且比预估好。

## 四、创新方向判决

### 被否:早 miss 门控(用户原始假设,负结果决定性)

census Ceager 真实口径(热树+并发 3,n=723 summary;热树下 **97.4% 的步带草稿**,机制上允许的"无草稿步"在热树里罕见):

**判决表:"前 N 个有草稿步全 miss"预测"整请求低命中(<20%,约后 1/4 分位)"**

| N | 触发数(占比) | precision | recall | 触发组命中率中位 | 未触发组 |
|--:|---|--:|--:|--:|--:|
| 2 | 159 (22.0%) | 0.14 | 0.35 | 25.4% | 27.0% |
| 3 | 50 (6.9%) | 0.24 | 0.19 | 23.2% | 26.8% |
| 4 | 27 (3.7%) | 0.26 | 0.11 | 23.0% | 26.7% |
| 5 | 13 (1.8%) | 0.31 | 0.06 | 22.3% | 26.7% |
| 6 | 8 (1.1%) | 0.25 | 0.03 | 22.3% | 26.7% |

(名词:**触发组** = 满足报警条件"前 N 个有草稿步全部被拒"的请求(触发数=其个数);**未触发组** = 其余请求;末两列是两组的**整请求命中率中位数**(不是低命中占比)。若要看"组内低命中占比":触发组=precision 列,未触发组约 7-8%。**措辞修正(7/6,经用户追问)**:低命中基率实为 **8.7%**,故 N=4 的 precision 0.26 是 **3× 富集——信号并非零**,原"≈随机抓"说重了;但判决不变,支柱换为:①recall 仅 0.11(漏 9/10 真输家) ②触发组整体仅低 ~3pp(74% 是误伤) ③收益上限归零(N=4 可省步 4.2% 且多为误杀)——**信号"弱而无用",非"不存在"**。)

**N 加大扩展(7/6 补,经用户追问)**:N=8 触发仅 4 个、N=10 剩 1 个、N≥16 为 0——信号蒸发。**根因量化**:各请求"单步中签率"近同质(中位 37%,p10-p90 28-48%),同质 p=0.37 下 P(连续 N 全 miss)=0.63^N 几何塌缩(N=8→2.6%)——**大 N 无人触发,小 N 触发的是运气差的正常请求**,两头堵死。收益上限同样归零:N=2 全关只碰 25% 的步且 86% 误杀,N≥6 可省步 ≤1.2%。

附加判据:
- **迟到爆发 0/723**(定义:开局下注全 miss、生成中途兄弟输出入树"救兵赶到"后命中率起飞的请求——判据"前 1/3 几乎全 miss 且后 1/3 命中>30%";若存在则一刀切关会误杀、需"关后可重开"探针层。实测不存在:prompt 树保底+请求同质,没有"开局全灭等救兵"的剧本)→ 探针重开层不需要;
- ctx 四分位命中率完全持平(26.7/26.0/27.0/27.2%)——replay 的"长 prompt 必亏"被并发+热树摊没。

**幸存信号只有 tag 静态先验**(report/query_plan 满押——但这是默认行为,无增量创新)。范围限定:判决属"热树+并发+中 ctx"生产口径;冷启动/超长 ctx 场景另论。

### 改立:命中率天花板(草稿源质量,D 系列)

summary 命中率中位 26.7%、天花板参照 report 59.5%——**若草稿源(题型感知/兄弟互抄感知)把 summary 抬到 40-50%,tokens/step 1.4→2+,summary 1.07x→1.3-1.5x**。晨间 A2(题型×命中率排行+q17/q1 尸检)为其选题。

### 新增:草稿费 7.5 的结构拆解(潜在系统优化)

若"激活费"成分大(晨拆),则"降低草稿激活成本"是新的系统标的,与草稿源质量正交。
