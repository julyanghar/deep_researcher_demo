## 二、三个 delta

**① 投机效应 c1→c2(⚠️ 含 eager 混淆:vanilla 是 cuda-graph、Ceager 是 enforce-eager)**
- e2e **4531→3233 = −29%**;几乎全靠 **report 段减半**(2176→1080,−50%);summary 段仅 −9%(1371→1248)。
- → **投机价值 ∝ 照搬内容**:report 照抄多(命中 59.5%)大赚,summary 改写(命中 26.7%)沾不上。

**② 并行效应 c2→c3(旧 prompt)**
- e2e **3233→4155 = +29% 更慢**;**summary 段 1248→1962(+57%)**、report 段 1080→1179(+9%)。
- 输出爆炸:sumTok 173.7k→395.1k(**2.27×**)、repTok 100.5k→174.8k(1.74×);调用数 summary 315→1200、report 20→147。
- → 旧 prompt 并行把输出撑爆,净拖慢。

**③ ★prompt 效应 c3→c4(同 server,关键)**
- e2e **4155→3626 = −13%**;**report 段 1179→672(−43%)**、repTok 174.8k→92.8k(−47%);summary 段 1962→1938(**−1%**,减 token 救不了饱和,sumTok 却 −27%)。
- **逐题 17/20 更快、中位 0.80×**(见 §四)。
- → 去冗余 prompt 有真实 e2e 价值,**几乎全来自 report 段**。

**关键横比 c2 vs c4(该不该并行,用最好 prompt)**:e2e 3233→3626 = **+12% 更慢**;summary 段 **+55%**(1248→1938)、report 段 **−38%**(1080→672)。→ **report 并行赚、summary 并行亏,两个都开净亏 12%。**

**逐题读法**:

- **并行 vs 非并行(c4 vs c2)**:**双峰**——省的题(q4/q11/q20 ~0.75×)是 report 重的题(并行填空闲有肉);亏的题(q8 2.11×、q15 2.08×、q2 1.56×)是 summary 重/report 轻的题(并行只加饱和的活)。**均值 +12% 慢 = 这两拨的合力。**

## 五、结论

1. **投机 = 大赢(−29%)**,靠 report(照搬多);summary 沾不上——**投机价值 ∝ 照搬内容**。
2. **并行:report 赚(报告段 −38%)、summary 亏(摘要段 +55%)**;两个都开比非并行 **+12% 慢**。最优 = **投机 + 只 report 并行 + 新 prompt**。
3. **去冗余 prompt 真实省 13%(逐题 17/20)**,几乎全来自 report 段(−43%);summary 段减 token 但墙钟不降(饱和)。
4. **诚实口径**:c1↔c2 混了投机+执行模式两项(不是纯投机);summary 并行拖累已被 20 题坐实,早先"summary 并行反而快"(基于并发8突发 replay)**不代表真实管线**,以本 20 题 e2e 为准。
