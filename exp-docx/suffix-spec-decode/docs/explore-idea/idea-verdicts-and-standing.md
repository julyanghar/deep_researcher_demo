## 一、已处置的 idea(9 条否决,每条挂判决书)

### 1. 早 miss 门控(自适应关投机)❌
- **原设想**:投机头几步连续 miss → 判定该请求不适合投机 → 关掉省按量费。
- **判决**:生产口径(热树,n=723 summary)信号无效——"前 N 个有草稿步全拒"预测低命中的 precision 0.14-0.31 ≈ 基线;触发组命中率 23-25% vs 未触发 27%,不分离。热树下冷启动本身罕见(N=4 仅触发 3.7%)。
- **方法注解**:①**生产口径** = census 条件(e2e 40 题、题内并发 3、**热全局树**——树里积着此前所有请求的输出,与真实部署一致;对照 replay 冷树显微镜口径——早 miss 在冷树下可能真有效,但那不是部署形态);**n=723** = 40 题的全部 summary 请求数("有草稿步≥6"的防御性过滤实际排除 0 个——热树下 **97.4% 的步都带草稿**)。**"草稿步"定义**:每步 proposer 拿上下文结尾(≤24 token)在两棵树(自己 prompt+全局树)里找匹配,**匹配门槛低到"最后 1 个 token 在树里出现过"**——匹配上通常就发草稿(匹配越长预算越大;例外:匹配到但所有续写概率都低于 min_token_prob 时发 0 注,故 dr>0 占比是匹配率的**下界**),该步即"有草稿的步";dr=0 步实测仅 2.6-2.7%。**三级漏斗**:匹配(≥97.4% 的步,按请求 100% 能匹配,不稀缺)→ 下注(均 2.9/步,66% 仅 1-2 token)→ 命中(26.7%,**这才是稀缺资源**,创新收敛到抬命中率而非匹配率的原因)。**复核(经质疑重验)**:只算 SUMMARY 97.3%;逐请求中位 97.7%、最低 88.5%;冷树期(最早 30 个 summary)也有 94.9% → **"步步下注"的主力是每请求自己的 prompt 树**(prompt 里几千 token 搜索结果,结尾 token 几乎总能匹配),全局树只从 94.9% 抬到 97.9%——**树热改变下注质量(命中率),不改变下注频率**。草稿构成:66% 的注只压 1-2 个 token(dr=1 占 29%、dr=2 占 37%、dr≥6 仅 6%)——低门槛+弱证据海量下注,正是"97% 步步下注但命中率 26.7%"的机制解释。②**precision 口径**:N 扫 2..6 五档;信号 = 前 N 个 dr>0 的步其下一步 acc **全为 0**(全 miss 才报警;第 k 步草稿的命运记在第 k+1 步采样里);真值 = 整请求 Σacc/Σdr < 20%(约后 1/4 分位,阈值预先定);precision = TP/(TP+FP)。实算例 N=4:触发 27 个,其中真低命中 7 → precision 7/27=0.26、recall 7/64=0.11;而全体低命中基率本就 ~15-20% → **报警组 ≈ 随机抓,无区分度**。
- **大白话五理由(每条一个致命数字)**:
  1. **报警器弱而无用**(措辞经追问修正):基率 8.7% 下 precision 0.26 有 3× 富集——信号非零;但 recall 仅 0.11(漏 9/10 真输家)、报警组整体命中率只比大盘低 3pp(74% 误伤)——**抓得又少又不准,不值得为它动手**;
  2. **根子是数学——大家的硬币一样歪**:722 个请求的单步中签率挤在 28-48%(中位 37%),近同质;头几步全 miss 只说明"这几把运气差"不说明"这请求命不好"。同质性把 N 两头堵死:N 大→0.63^N 塌缩(N=8 仅 4 人触发,N=16 为 0),N 小→86% 误杀。**没有一个 N 能用**;
  3. **没有目标人群**:命中率分布紧(20-36%)无极端烂请求,且热树 e2e 下 summary 整体已 1.07x 净赚——要救的人不存在;
  4. **白送完美信号也没钱省**:eager 下按量费仅 0.62ms/位,激活费 5.8 批级共享关单请求躲不掉;N≥6 可省下注步 ≤1.2% 的全体占比;
  5. **没有需要保护的逆袭剧本**:"开局全灭等救兵"的迟到爆发请求 0/723——prompt 树保底让人人开局就有 ~37% 中签率,连"关后重开"机制都无需设计。
  **一句话**:这是"人人中等运气"的赌局——没有可识别的倒霉蛋(②③)、报警器认不出人(①)、抓到也没钱省(④)、没有逆袭者要保护(⑤)。
- **判决书**:[overnight-verdicts §四](../examine-spec-tax/overnight-verdicts-2026-07-06.md)(含 N 扫至 20 蒸发表/同质性分布/迟到爆发判据)。
- **复活条件**:冷启动为主/低并发/无全局树/prompt 极短的部署形态(①②⑤ 的前提在那里不成立;非本项目场景)。

### 2. ctx/长度门控 ❌
- **原设想**:长上下文时草稿验证贵(里程费)且 replay 显示长 prompt 必亏 → 按 ctx 收手。
- **判决**:e2e 命中率按 ctx 四分位完全持平(26.7/26.0/27.0/27.2%),tok/s 亦平——里程费被并发+摊薄稀释,replay 的 ctx 效应是 batch=1 伪影。
- **判决书**:overnight §四;replay 侧原始现象(六修斜线)在 [how-we-found §7.3b 六修](../examine-spec-tax/how-we-found-the-fixed-tax.md)。

### 3. 题型感知调度 ❌(收敛,非全灭)
- **原设想**:某些题型易照抄 → 分题型调 k/开关。
- **判决**:方向存在但幅度小——题级命中率仅 20.3~34.0%(±7pp),cov12 照抄率 ρ=0.374(中弱)。定性画像成立(整理/清单型高、论述/人文型低)但不足以支撑调度收益。
- **判决书**:[overnight-verdicts §四b](../examine-spec-tax/overnight-verdicts-2026-07-06.md)(含 40 题排行头尾对照)。

### 4. 兄弟互抄感知的草稿源增强 ❌ 定案(2026-07-06 决定性实验,经 ⚠️ 后重新判死)

**决定性实验(全新树串行 replay,测接受率 vs 完整兄弟数,同轨迹口径)**:接受**率**对兄弟数**完全平**——0 兄弟 29.2% → 满兄弟(10+)29.5%,Δ+0.3pp,全程 28-31% 噪声。**兄弟连"全预热"最好情况都抬不了接受率**。这一锤定音,推翻了上一版据 v2"42.5%"做的 ⚠️ 降级(那 42.5% 确系测量假象)。
- **一个诚实的温和补充**:兄弟确实让**每步草稿数**涨(2.06→2.55,+24%,料多→匹配长→预算大),这些多草稿在同样 29% 率上命中 → 每步 token 温和涨 **+9%**(1.60→1.75),**3-5 个兄弟即饱和**。所以兄弟有个**被动小利**(+9% tokens/step,经"更多草稿"非"更高率")。
- **根因(token 口径实测,2026-07-06):"兄弟可抄内容"本就 ≈0**。此前当作"兄弟有肉"的"union 覆盖 31.7%"是**字符块**口径(同题实体/术语短片段);换 **token 长段口径,兄弟 summary 输出↔输出 cov24 仅 0.69%(中位 0.00%)、cov8 4.48%**,和输入↔输出 cov24 ~0.4% 几乎一样低。**字符级"看着像"、token 级"没得抄"**——所以增强无肉,与接受率 flat 同一根因。
- **为什么增强仍判死**:①率天花板 29% 是**内容可抄性**决定的、非树组织方式决定,增强顶不动;②那 +9% 被动树已吃满、且早饱和。增强的头上空间 = 可抄上界 − 被动已吃 ≈ 0。
- 早先的旁证(census 密兄弟 28.3% ≈ 疏兄弟 26.4%、+1.8pp 边际)方向全部一致,现由控制实验坐实。
- **想做什么**:同题 3 个 researcher 并行,输出互相大段撞车(T3:互抄可收割中位 18.8%,并入兄弟输出可让可抄池近翻倍)。全局树已**被动**实现互抄(完成即入树,后来者自动能抄);本 idea 想**主动增强**——给兄弟输出更高匹配优先级/更深索引/更大预算,或调度上让兄弟先完成喂树。
- **中间的弯路(留档,已被决定性实验取代)**:曾据 +1.8pp 判 ❌ → 用户质疑 → 据 v2"26.4→42.5 翻倍"降 ⚠️ → 发现 42.5% 是 v2 手工切窗混进 report 的测量假象(轨迹法验证 census summary 干净口径 28.3%)→ 决定性实验坐实率对兄弟数全平。详见 [questions-log-2.md Q3](../questions-log-2.md)(含轨迹法 vs server 窗口法的口径澄清)。
- **复活/证伪条件**:接受**率**平是内容属性,除非换更高雷同度的负载(同题输出逐字重合率远超 DRBench 的 ~31% union),否则不复活。
- **判决书**:overnight §四b 关联表;互抄现象本身的量化(union 边际 +12.9%)在 [progress-report §3.6](../examine-spec-tax/tax-split-progress-report.md)。

### 6. suffix 强开异步调度 ❌
- **原设想**:异步被关是 suffix 的结构损失 → hack 白名单拿回来。

### 7. k8/pad 配置学 ❌
- **原设想**:截短草稿省按量、padding 定形状进 graph 桶 → 部署甜点。
- **判决**:整套只在"混合模式+batch=1"语境有意义。全 eager 下 k 无所谓(33.7≈33.9)、pad 倒亏 3.5ms(2×2 闭环);生产口径 pad 是负资产(G-e2e 0.97x,CeagerK8pad 双输)。

### 8. 迟到爆发探针(关后重开)❌

- **判决**:"前 1/3 全 miss 后段翘头"的请求 **0/723**——现象不存在,热树暖场先于一切。
- **判决书**:overnight §四。

### 9. 命中信号做剪枝/门控 ❌(信号真、但省开销两条路都不成立;2026-07-06)
- **原设想**:用"投机是否命中"当信号——某段 draft 命中过之后还命中、否过之后还否 →(a)剪掉注定否的 draft 省 decode 开销,(b)按命中/否**保留或驱逐 KV** 减注意力开销。
- **信号本身实测成立(且更正了我早先的步级误判)**:**内容级**——同一段 draft(matched-suffix+提议 token)重复出现时,**首次命中→后续命中 73.6%、首次否→后续命中仅 10.7%**(基线 32.4%),73.9% 的重复 draft 自始至终同结果。所以"命中/否高度可复现"是真的。(这是**内容级持久性**,区别于步级"连续 miss 会在进下个可抄区时恢复 ~10%"的另一问题——早先我拿步级口径答成了这题,已更正。)
- **(a)剪 draft 省开销 = 净亏**:注定否的 draft 本就短(1-2 位,eager 0.62ms/位),正确剪省芝麻;11% 翻盘率下每误剪一个"其实会命中"的 draft → 丢整步加速 ~24-40ms;EV≈0.89×1 − 0.11×32 = **负**,可省上界 <3% decode。且否过的 draft 的 KV 本就没留(拒绝即弃),"驱逐 miss KV"是已有机制、零增量。
- **(b)命中信号驱逐 KV = 信号错配**:KV 该不该留由"未来会不会被 attend"决定(H2O 注意力信号);命中衡量的是"表面可抄"——两者正交,会误杀"重要但非逐字"的上下文伤质量。且 batch=1 时注意力才占每步 ~0.8%(summary)~7.5%(report),没多少可省(见 anatomy)。
- **信号唯一正解用法**:拒绝记忆做 **re-rank**(否过就改提第二名)以抬命中率——归机制侧(§二.4),是进攻不是省开销。
- **物证**:内容级复现 + 经济账 `~/modify-code-runs/suffix-spec-decode/mechanism-sim/{draft_reproducibility,gating_economics,hit_persistence}.py`;计算解剖 [decode-step-compute-anatomy.md](../examine-spec-tax/decode-step-compute-anatomy.md)。
- **复活条件**:无(经济不对称是结构性的:省短草稿费的芝麻、赌整步加速的西瓜)。

**共同教训**:热全局树 + 题内并发把大部分"聪明调度"的空间提前吃掉了——系统本身已经处在接近"自动最优调度"的状态,剩余变差要么太小(题型 ±7pp)、要么不可预测(命中率同质)。**唯一例外是 #4(兄弟增强)**:它没被吃满,只是被我用错口径误判过一次——真要判死需专门消融实验。

## 二、立住的贡献(4 条,每条挂物证)

### 1. ★混合执行锁竞争的发现 + 一行修复(已兑现,最强贡献)
- **发现**:vLLM 投机的 graph+eager 混合模式下,发射线程停在 `cuLaunchKernel` 内部驱动私有条件变量上 **4.5ms/步**(调用栈:`pthread_cond_timedwait ← libcuda ← cuLaunchKernel`);graph 段与 eager 发射交错提交触发驱动内部串行化;全 eager 对照 0.00s。
- **修复与生产验证**:`--enforce-eager` 一个参数 → 40 题 DRBench 端到端**中位 +28%**、分段 tok/s 全正(SUM 1.07x/QP 1.54x/REP 2.37x)、**decode 口径零回归**(仅有的 2 个墙钟落后题经尸检为 agent 路径方差)。
- **物证**:[how-we-found §7.3b-终审+七修](../examine-spec-tax/how-we-found-the-fixed-tax.md);生产数字 [overnight §二+§四b](../examine-spec-tax/overnight-verdicts-2026-07-06.md)。

### 2. 五常数成本模型(投机解码的可加成本解剖)
- **模型**:每步 = graph 基线 24.0 + eager 附加 0.5 + spec 管线 1.9 + 混合边界税 8.1 + 草稿费(激活 5.8 + 0.62/位置)。
- **强度**:7 个配置预测=实测、残差 0;经历两道预注册预言题的淘汰修正(⑥预测 0.9→实测 0;C0eager 预测 28.1→实测 26.4,每次错误都定位出记账错误)。
- **物证**:[cost-anatomy §二](../examine-spec-tax/suffix-vs-native-cost-anatomy.md)。

### 3. 草稿激活费之谜(新系统标的,未解)
- **现象**:"这一步有草稿"本身收 5.8ms 激活费(与草稿多长几乎无关,单价仅 0.62/位置);其中 CPU 机件 1.26 已归因,**余 ~4.5ms 未归因**——税表最后一块砖。
- **若解决**:summary 每步 33.9→~29.4,单独再 +15%;叠加在已有 1.07x 上 → ~1.23x。
- **嫌疑清单**:有草稿时的 logits 多行处理路径/spec 采样 GPU kernel/输出拷贝路径切换。
- **物证**:cost-anatomy §五;拟合数据 overnight §六.2。

### 4. 机制侧命中率抬升(草稿源的正确打开方式)
- **依据**:同一棵树在 report(结构化长输出)上命中 59.5%、summary 26.7%——差距在**内容形态**不在树;题型/互抄调度已判死,剩余空间在机制:更长匹配窗口、更好置信打分、summary 的结构化引导(让输出更"可抄")。
- **注意**:预期要诚实——论述型内容 ~20% 可能是物理上限,抬升目标定在整理型题的 34%→45% 区间更现实。
- **物证**:overnight §四b"对 D 系列的修正"。

## 三、当前推荐的 idea 优先级(待拍板)

1. **激活费 4.5 归因**(§二.3):系统侧,方法论现成(闭合秒表/nsys 全开已验证过打法),预计 1-2 天出定案;解决即直接换算成吞吐;
2. **机制侧命中率**(§二.4):偏算法,和 KVCOMM 系工作衔接自然;先做"结构化引导"小实验(prompt 层让 summary 输出更模板化,量命中率变化)成本最低;
3. **并发下 KV pruning(系统侧,新候选,2026-07-06 记)**:批处理把权重读摊薄、但 **KV 各读各的** → **summary 段 ~9 路并发时 KV 占每步访存 ~20%**(vs batch=1 的 2.7%),砍 50% KV 可省 **~10%**(现实 ~5-8%,随并发浮动)summary decode;**report 单流不吃这红利**(停在 3.8%)。**三要点**:①用**注意力信号(H2O)非命中信号**(命中信号已判死,§一.9);②summary 的 KV 是要总结的**源文档**、是 pruning 的**质量困难案例**(注意力铺开、须验);③是**独立的长上下文/高并发 KV 压缩项目**,与投机/深注扫描**正交**(一个砍每步注意力、一个砍步数)。**注**:此为系统侧吞吐优化,不属"抬命中率"线;是否值得做取决于 summary decode 占端到端比例 × 实际并发。物证:[decode-step-compute-anatomy.md §五-六](../examine-spec-tax/decode-step-compute-anatomy.md)。
4. **论文合订**:①+②任一有果后,故事线 = 锁竞争发现+修复(主) + 五常数模型(方法) + 判决书(分析/负结果,对话 adaptive-spec 文献)。
