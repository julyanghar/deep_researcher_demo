# 提案:机制侧命中率抬升(草稿源的正确打开方式)

> **地位**:idea 总卷 §二.4 的展开;判据预注册,实验未排,等拍板。
> **一句话**:五常数模型改写了下注经济学——eager 下一个草稿位置只值 0.62ms、盈亏线=单位置接受率 ~2.4%,而实测 position 10-12 仍有 5-9% 接受率 → **当前策略下注太浅**;旧"调参无用"结论是在混合模式(位置贵 1.4-3ms)下得出的,需在 Ceager 经济学下重验。

## 一、为什么"机制侧"是对的战场(证据回顾)

1. 三级漏斗:匹配 ≥97%(满了)→ 下注均 2.9/步、66% 仅 1-2 token(**浅**)→ 命中 26.7%(稀缺);
2. 同一棵树在结构化内容上能到高命中(report 59.5%、query_plan 39.4%)——瓶颈在内容形态与下注策略,不在树;
3. 调度类方案(题型/互抄/门控)已全部判死(见 [idea-verdicts-and-standing.md](idea-verdicts-and-standing.md) §一)。

## 二、核心命题:下注经济学已因成本结构改变而反转(可证伪)

- **成本**:加深一位置 = 0.62ms(eager 实测,pad 反推+四点拟合);混合模式下同一位置实际 1.4-3ms(桶抖动按形状收费)——**旧参数扫描(prob 0.05-0.5/factor,判"翻不正")是在贵位置的经济学下做的**;
- **收益**:一个被接受 token ≈ 省一整步 ≈ 26-34ms → 盈亏线 = 单位置接受率 ~**2.4%**;
- **实测尾部**(server 日志逐位置接受率):p4≈18%、p8≈12%、p10-12≈5-9%——全部高于盈亏线;
- **诚实警告(幸存者偏差)**:现有尾部率来自"树自信才发那么长"的样本;强行加深的新位置接受率必然更低。命题的判决交给实验,盈亏线低(2.4%)是容错垫。

## 三、三条路线与预注册判据

### 路线 1:深注扫描(最便宜,先做)
- 旋钮:`max_spec_factor` 4→8→16;`min_token_prob` 0.1→0.05→0.02;`max_tree_depth` 24→48(旧计划 depth 扫描,一直未跑);
- 实验:Ceager 上网格 replay40(主)+ 最优点 10 题 census 复核(防 batch=1 伪影);
- **判据**:tokens/step 与 char/s **双涨**为胜(单涨 tokens/step 而 char/s 平=白验证,判负);预期 tokens/step 1.43→1.6+,summary 1.07x→1.15-1.25x;
- 风险:并发下位置单价略高于 0.62(批共享稀释),census 复核兜底。

### 路线 2:结构化引导 2.0(带 L3 尸检报告重做)
- 依据:结构化内容命中翻倍(39-59% vs 26.7%);
- **L3 死因直查**:机制成立(吞吐 1.49x)但输出膨胀 +72% 吃光收益、1/10 撑爆上下文 → 新设计约束:**模板化格式(编号事实条目/禁散文胶水)+ 硬长度上限(max_tokens 与 prompt 双控)**;
- 实验:改 summary prompt 模板,10 题对照(同天 Ceager);
- **判据(三条同时满足才算活)**:接受率 ↑≥5pp;长度膨胀 <10%;质量平(kimi-k2.5 盲判,沿用 supervisor 实验口径)。

### 路线 3:换草稿源(EAGLE-3 对照;大工程,1/2 摸顶后再议)
- 逻辑:检索式草稿吃"逐字重复",模型式草稿吃"分布相似"——paraphrase 型 summary 是后者主场;
- 定位:天花板探测 + 论文对照基线;工程量大(draft model 部署/训练适配)。

## 四、预期与边界

- 三路线正交,可叠加;合计现实目标:summary 1.07x → **1.2-1.35x**(整理型题冲 1.4+);
- 边界:论述型内容 ~20% 命中可能是物理上限(A2 画像),不指望人文题翻身;
- 与系统线(激活费 4.5 归因)独立并行,互不阻塞。
