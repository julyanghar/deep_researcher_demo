## 1. suffix spec vs vanilla 的 spec tax(固定税)分析

之前 suffix spec 相比 vanilla 慢在哪、做了什么优化。


| 常数 | 值 ms/步 | 什么时候存在 |
|---|--:|---|
| ① vllm 基线 | 24.0 | 每步（读一遍 32B 权重 + 整图回放） |
| ② eager 附加 | +0.5 | 用 `--enforce-eager` 时（放弃整图的净代价） |
| ③ spec 管线 | +1.9 | 投机开启固定的流程（propose+采样回传+memcpy+GIL） |
| ④ graph+eager 混合 | +8.1 | 仅 graph+eager 混合模式交；主体=**驱动锁等待 4.5**（+图段启动 1.4+杂项 2.2）；**全 eager 一刀消掉** |
| ⑤ 尚未归因 | 固定值 5.8 ms + 0.62×草稿位置数 | 有draft就存在 |

## 4. 连续 miss 门控 (自适应关投机) 否决
- **原设想**:投机头几步连续 miss → 判定该请求不适合投机 → 关掉省按量费。
- **判决**:生产口径(热树,n=723 summary)信号无效——"前 N 个有草稿步全拒"预测低命中（整体命中率小于15%）的 precision 0.14-0.18。热树下冷启动本身罕见。
- 全局树条件下，suffix spec 几乎占据绝对优势：2/40 lose。

## 6. Eager vs Suffix

AL = 一次大模型前向吐几个 token(1 个必得 + 接受的草稿数)。即每步收入
每步成本是"支出"
加速比 = 收入 ÷ 支出涨幅。

| | 每步成本 | AL(每步收入) | 每 token |
|---|--:|--:|--:|
| suffix(Ceager)summary | 34.4ms | **2.77** | 12.4ms |
| eagle3(k3)summary | 34.0ms | 1.41 | 24.1ms |
| suffix report | 46.0ms | **4.61** | 10.0ms |
| eagle3 report | 43.3ms | 1.58 | 27.4ms |


| k | summary 接受率 | report 接受率 |
|--:|--:|--:|
| 1 | 29% |  37% |
| 3 |13.6% | 19.3% |
| 5 | 8.4% | 12.8% |

## 7. paper 故事

1. 任务分两类:**照抄类**(输出与输入重叠大)/ **分析总结类**。
2. 照抄类 → 直接用 **suffix 投机**(效果明显);分析类 suffix 效果差 → 用 **EAGLE-3**(自训或现成)。
3. 加**判决模块**:判这一次 LM 调用是照抄还是分析。可用 **Qwen3-32B + 小 LoRA** 或 **MLP** 训练,信号简单:**输入 = 大模型 hidden state,输出 = 输入与输出的覆盖率**。
